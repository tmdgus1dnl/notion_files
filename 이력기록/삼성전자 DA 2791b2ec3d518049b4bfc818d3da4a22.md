        ## 📌 1. 슬립 모드 알고리즘 (Sleep Mode / Duty Cycling)
        
        **개념**
        
        - 센서나 무선 통신 모듈은 항상 켜져 있으면 불필요한 전력 낭비.
        - **필요할 때만 켜고, 나머지 시간에는 슬립 모드(저전력 대기)로 두는 방식**.
        - Duty Cycle = (동작 시간 / 전체 시간). Duty Cycle이 낮을수록 전력 절감 효과.
        
        **적용 사례**
        
        - IoT 센서(온도, 습도 등) → 1분마다 측정, 나머지 59초는 sleep.
        - 가전에서도 Wi-Fi 모듈, BLE 모듈 같은 통신 장치에 많이 활용.
        
        **내가 할 수 있는 개선 포인트**
        
        - 네트워크 서버 구현 경험(HTTP 서버) → **비동기 요청 처리 로직**을 알고 있음 → 이벤트 기반으로 wake-up 트리거를 설계 가능.
        - 즉, “항상 대기” 대신 “이벤트가 올 때만 깨어나는 구조”를 구현할 수 있음.

        ## 📌 2. 실시간 스케줄링 알고리즘 (Real-Time Scheduling)
        
        **개념**
        
        - 가전은 실시간성 요구: 예) 세탁기 모터 제어, 냉장고 온도 제어.
        - OS는 여러 태스크를 동시에 돌려야 하므로, 어떤 태스크를 언제 실행할지 정해야 함.
        - 대표 알고리즘:
            - **Rate Monotonic (RM)**: 주기가 짧은 태스크에 높은 우선순위.
            - **Earliest Deadline First (EDF)**: 마감 시점이 가장 가까운 태스크를 우선 실행.
        
        **적용 사례**
        
        - 모터, 펌프, 센서 데이터 처리 → 기한 내 실행 실패 시 품질 문제 발생.
        
        **내가 할 수 있는 개선 포인트**
        
        - 42서울에서 멀티스레드 동기화, 교착 방지 구현 → **리소스 경쟁 해결 경험**.
        - 이를 기반으로 “실시간 태스크 충돌”을 예방하는 최적화 로직 설계 가능.
        - 예: 냉장고 온도 센서 데이터와 디스플레이 업데이트가 동시에 리소스를 요구할 때, 우선순위 부여로 안정적 실행 보장.

    저는 임베디드 소프트웨어 개발에서 가장 중요한 역량은 안정성 확보와 최적화라고 생각합니다.
    특히 저의 집에서 사용하는 삼성 가전이 24시간 구동되는 환경에서도 안정성과 효율성을 유지하는 모습을 보고 이런 부분을 느꼈고,
    여러 프로젝트를 수행하며 이러한 역량을 입증해 왔습니다.
    
    병렬 프로그래밍 프로젝트에서는 초기 전역 Lock 방식으로 인해 인스턴스가 교착 상태에 빠지거나 성능이 저하되는 문제가 있었습니다.
    이를 해결하기 위해 자원별 Lock 분할과 대칭 깨기 알고리즘을 적용했고,
    이를 고 경쟁 시나리오에서 벤치마킹한 결과 수행 시간을 150초 -> 75초로 단축해 약 2배의 성능 개선을 달성하는 동시에, 교착 상태를 완전 방지하는 성과를 냈습니다.
    
    또한 Linux Shell 구현 프로젝트에서는 잘못된 입력으로 인한 셸 비정상 종료 문제가 발생했습니다.
    이를 개선하기 위해 AST 트리 구조를 기반으로 에러 격리와 상태 복원 구조를 설계해,
    잘못된 명령에 해당하는 자원만 회수하고 이후 명령은 정상적으로 이어지도록 구현했습니다.
    그 결과 크래시 없이 안정적인 연속적 명령 실행이 가능해졌습니다.
    
    마지막으로 웹서버 개발 프로젝트에서는 대규모 요청으로 인한 부하 문제로 서버가 불안정해지는 상황이 발생했습니다.
    이를 개선하기 위해 이벤트 기반 논 블로킹 서버를 구축하고, HTTP/1.1 프로토콜의 상태 코드, 타임아웃 처리를 철저히 준수하며
    벤치마크 모드에서 최대 속도로 부하를 걸어 10분 이상 가용성 99.5% 이상을 달성하는 성과를 냈습니다.

    24시간 구동 환경에서도 예측할 수 있는 성능과 높은 신뢰성을 보장하는 시스템을 구현하는 데 일조하겠습니다.
