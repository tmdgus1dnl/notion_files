# Get Next Line

설명: 파일 입력 처리
상태: 완료

파일 디스크럽터(fd)에서 한 줄씩 데이터를 읽어오는 함수를 구현하는 과제이다.

보너스 파트는 여러 파일(여러 fd)에서 동시에 반환 작업을 수행할 시에

현재 파일이 아닌 다른 파일에 get_next_line함수를 사용하고

다시 현재 파일로 돌아왔을때

현재 파일을 어디까지 읽었는지 기억하는 기능까지 구현해야 한다.

# 몇가지 함수 설명

realloc :

원래 있었던 str의 문자열을 새로운 new_str에 복사,

그 이후 다음 buff 붙이는 작업 진행, 이후 new_str 반환

get_buff : 

static변수를 이용해 buff안에 \n다음의 문자열을 기억하여 str앞부분에 삽입해야함.. 어떻게?

static buff[BUFFER_SIZE] 이렇게 정적 할당을 한 후에 

이미 버퍼 안에 있는 문자열에서 \n이후의 문자열의 길이를 세고 

newbuff 할당후에 개행 이후 문자열 삽입, 그 후 buff에 read 수행.

# 리스트 반복문 구조

리스트 역순 탐색 = 이중 연결 리스트로 해결

ret_str = 리턴할 문자열

node_fd = 파일 디스크럽터 정수

next_str = 다음 노드로 전해줄 문자열

ret_str 엔 첫 개행 이전의 반환할 문자열이 저장되어 있다.

next_str 엔 첫 개행 이후 문자열이 저장되어 있다.

현재 노드의 next_str안에 개행이 저장되어있다면 read를 하지않고 개행 전까지의 문자열을 다음 노드의 ret_str에 저장

다음 노드의 next_str에는 개행 이후의 문자열을 저장.

현재 노드의 next_str에 개행이 저장되어 있지 않다면 

현재 노드의 next_str을 다음 노드의 ret_str 앞에 넣어주고

개행이 나올때 까지 read 를 수행하며 다음 노드의 ret_str에 계속 붙여준다.

작업 이후에 현재 노드의 next_str은 free = 메모리 누수를 방지하기 위해

보너스 같은 경우는 prev 멤버 변수를 이용해 리스트를 역순 탐색하여

해당하는 노드의 node_fd가 fd와 같을시 반복 멈추고

위에 기재한 반복을 시작.

# 코드를 짜며 느낀점

static은 = 0 으로 왠만하면 초기화 시키지 말자.. 주소를 기억하기 위해 static으로 사용한거니깐.

애초에 선언할때 0으로 초기화 해준다..

리스트 함수 쓸때 더블 포인터로 리스트 주소를 받아서 사용하는 순간이 굉장히 많다.

어떤 함수는 처음 들어온 주소를 저장해놔야 다시 반환할때 정상적으로 돌아가고

어떤 함수는 저장하지않고 반환해야 정상적으로 돌아간다.

때에 따라 맞는 방법으로 주소를 사용하면 좋을거 같다.

리스트 자체가 주소를 사용하는 순간이 많아서 그런듯. 이점 유의하자.

- get_next_line 함수가 끝나도 버퍼안에 여러 개행이 있을때 원하는 문장의 줄 또는 장소를 아는 방법
    
    노드 안에 next_str이라는 멤버 변수를 만들어 그 안에 현재 노드의 ret_str을 제외한 문자열을 다음 노드에 넘겨준다. 이렇게 하면 여러 개행이 있어도, 개행 앞의 문자열이 있어도 가져오는게 가능.
    
- 만약 static을 동적할당 한다면..? Free는 언제 해야하지..?
    
    해줄 필요가 없다. 애초에 동적할당 하면 메모리 공간은 계속 free를 해줘야 하고
    
    그러면 메모리를 가리키는 포인터만 static인 상태고 메모리는 계속해서 초기화됨.
    

# 디버깅 기록

빈 문자열이 substr에서 할당되어 널이 아니라 빈문자열이 반환되어 오류가 남 = substr애서 길이가 0이면 할당 전에 0반환하게 만듦.

해당 fd 파일안에 null이 들어가있으면 free를 시켜줘야 하는데 못해줘서 leak이 났다. = ret_str(반환할 문자열)이 아무것도 없으면 노드와 안의 문자열까지 전부 프리시켜줌.

strict 옵션에서 malloc protection 에러가 뜬다. 

일단 노드 몇개랑 그 밑에 문자열들이 프리가 잘 안되는듯. 테스트 케이스 살펴보자. = new가 널일때 리턴해버리는데 그럼 남은 리스트들이 해제가 안됬었음.

말록 터질만한 곳에 조건들을 걸어주니 안 터진다. 

말록이 터지면 널가드 뿐만 아니라 해당 노드까지의 같은 fd를 가진 노드들을 전부 해제 해야함.
