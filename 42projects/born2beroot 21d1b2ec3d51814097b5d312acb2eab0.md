# 가상화, 가상머신이란?

가상화란 하나의 물리적인 리소스(서버, 컴퓨터등)를 하나 또는 여러개의 가상환경으로 분리하여 동작하게 하는 기술.

가상머신이란 가상화를 통해 생성된 가상 환경. 자체 운영체제와 앱을 가지고있어 또 다른 독립적인 컴퓨터 시스템.

바이러스 프로그램 실험이나 다른 운영체제가 필요한 경우 유용하다.

# apt 와 aptitude의 차이점

둘다 Debian에서 패키지 관리를 위해 사용되는 도구.

apt는 명령어로 간단하게 패키지 관리가 가능하지만 aptitude는 텍스트 기반 UI를 제공하여 조금 더 복잡함.

apt보다 aptitude가 의존성 처리 와 우선순위 부분에서 더 엄밀하고 세밀하게 처리가 가능하고 여러가지 해결방법을 제시하여 사용자가 선택이 가능함.

따라서 aptitude는 대화형 도구이고 apt는 명령형 도구임.

# Rocky와 Debian 의 차이점

두개 다 리눅스 기반 운영체제 이지만, 몇 가지 다른점이 있다.

Rocky =  Cent OS Linux 가 Cent OS Stream 으로 전환되면서 Cent OS Linux 사용자들에게 원래 사용하던 진행 방향과 일치하는 대안을 제공하기 위해 만들어졌다.

Cent OS란 Red Hat Enterprise Linux (RHEL)의 소스 코드를 기반으로 하여 무료 및 오픈 소스로 제공되는 리눅스 배포판 중 하나이다. RHEL은 유료인 반면 이와 유사한 환경을 제공하면서 무료인 점이 다르다.

Debian = 독립적인 리눅스 배포판, 안정성과 자유 소프트웨어에 중점을 둔다.

여기서 자유 소프트웨어란 사용자가 소프트웨어를 실행, 복제 배포가 가능하고 소스코드에도 접근해 수정을 통한 개선과 학습이 가능 것을 말한다.

Debian을 선택한 이유 = Rocky는 기업의 안정성과 보안 요구에 따라 설계되어 초보자에게는 복잡하게 다가올 수 있어 사용자 친화적이고 자유로운 환경을 제공하는 Debian이 낫다고 생각하여 선택하게 되었습니다.

# 파티션과 LVM

파티션은 하나의 물리적인 하드디스크를 여러개의 논리적인 영역으로 나누는것을 뜻한다.

이렇게 나눈 영역을 여러 용도로 각자 사용 가능하다. 각 파티션은 독립적인 파일 시스템을 가질 수 있다.

파일 시스템이란 주로 파일과 디렉토리같은 계층 구조를 사용하여 컴퓨터의 데이터를 저장하고 관리하는 시스템

LVM = Logical Volume Manager

동적으로 디스크 공간을 조절할수 있어 여러 디스크 또는 파티션을 하나의 큰 저장공간(논리 볼륨으로 통합)으로 사용이 가능하고 하나의 디스크 또는 파티션을 여러개의 볼륨으로 나누어 사용할 수 있다.

스냅샷, 볼륨간 데이터이동,  효율적인 용량 관리, 볼륨에 새로운 하드디스크 추가를 통한 간편한 용량 확장등에 쓰인다.

마운트란? 예를 들면, 어떠한 저장공간이 있다. 특정 디렉토리를 설정하고 그 디렉토리를 이 저장공간의 마운트 포인트로 설정하면 해당 디렉토리를 통해 저장공간의 파일시스템에 접근이 가능해진다. 이 과정이 마운트이다.

결론 = 파티션이나 lvm을 설정할때 파일시스템(EXT4 etc..) 나 마운트 포인트(/, /boot etc..)을 설정하는것은 다 의미가 있었던것..

# SUDO

“super user do”의 약자로, 일반 사용자도 sudo를 사용하여 루트 권한이 필요한 명령을 수행할 수 있다. 

제한된 범위 내에서 원하는 유저에게 루트 권한을 부여 함으로써 시스템의 보안과 안전을 유지하면서 사용자에게 필요한 작업을 수행할 수 있도록 해준다.

일단 사용할때는 비밀번호를 입력해야하기 때문에 자신이 내린 명령을 한번 확인할 수 있다. 더블체킹.

로그가 남아 추적도 가능하다.

sudo정책 확인 → sudo visudo

# UFW

Debian 및 여러 리눅스 계열 운영체제의 방화벽으로 쓰임.

간단하고 사용하기 쉽기 때문에 사용자 친화적인 방화벽 도구.

하지만 고급 기능이 부족하고 복잡한 규칙을 적용할수 없어 보안 수준이 떨어질수 있다.

ufw는 iptables 방화벽 위에서 동작하는 프론트 엔드 도구이기 때문에 세부적인 고급 설정을 통한 보안 강화를 원한다면 iptables명령어를 사용하면 된다. ufw를 설치해도 같이 설치되기 때문.

# SSH

네트워크 통신을 안전하게 하기위한 프로토콜.

주로 원격으로 시스템에 접속하여 작업을 하기위해 사용함.

한 쌍의 키를 통해 인증함.

- Public key = 공개되어도 비교적 안전한 key 메시지를 전송할 때 **암호화**를 함 public key로는 **복호화할 수 없음**
- Private key = 절대로 외부에 노출되어서는 안되는 key 본인의 컴퓨터 내부에 저장 private key를 통해 암호화된 메시지 **복호화** 가능

# APPARMOR 와 SELINUX

apparmor 는 어플리케이션에 보안 프로필을 적용하여 해당 앱의 행동을 제어한다.

여기서 행동이란 어떤 파일이나 디렉토리에 대한 접근 또는 여러 권한을 규정할 수 있다.

또한 프로필 별 로그가 남아 이슈별로 추적이 가능하다.

selinux 는 미국 국가안보국에서 개발된 오픈소스 프로젝트이고, 대표적으로 RHEL에서 사용한다.

정책을 사용하여 접근을 제어한다.

파일, 프로세스, 네트워크등 리소스에 라벨을 부여한다. 정책을 통해 라벨 별로 각종 리소스의 여러가지 접근, 사용 권한들이 제어가 가능하다.

selinux는 정책을 어긴 동작은 차단된다.

apparmor는 프로파일을 만들어 권한을 규정하지만 말 그대로 권고 사항이라 준수하지 않아도 시스템이 실행될수 있다. 사용자가 규정을 어기는 동작을 해도 차단시키지 않고, 로그에 경고가 남는 수준. 이말은 강제성이 없다는 것.

# 포트포워딩

외부(로컬)에서 **어떤 서버 또는 IP(가상머신)로 접속할 때 특정 포트(4242)로 들어오는 트래픽을 전달해준다**.

외부 네트워크에서 내 가상머신에 접속하려고 하면 일단 Host PC인 클러스터의 Mac으로 접속해야 한다. 

Host IP 가 일치해야 하기 때문.

이 때 특정 포트(Born2beroot에서는 4242)로 ssh 포트를 열면 **포트포워딩**되어 바로 가상 머신에 접속할 수 있다.

또한 HTTP기본 포트인 80번으로 포트포워딩을 해놓으면 호스트에서 웹페이지를 요청하면 가상 머신의 웹 서버에서 80번 포트를 통해 웹 페이지를 전송한다.

# CRON 설정

cron = 특정한 시간에 혹은 특정한 시간마다 어떠한 작업을 자동으로 수행하게 하는 작업 스케쥴러

cron table에 사용자가 명령을 등록하면 cron데몬이 작업을 수행한다. 사용자마다 각자의 crontable 존재.

확인 → crontab -e

physical id = 물리적인 소켓를 나타낸다. 같은 id 번호의 cpu가 여러개 있으면 그 cpu들은 전부 같은 소켓에 있다. 

원하는 출력값은 물리적인 프로세서의 개수(= 소켓 개수) 이기 때문에 physical id 들을 sort 로 정렬하고 중복되는 행(id 값)은 -u로 제거한것.

vcpu는 가상 프로세서 개수이고 processor와 동일

free -m 을 쓰면 Mem이 있는 줄에 메모리 사용량, 총 사용량을 알수있고 이걸 바탕으로 awk printf $1,$2… 을 사용해 원하는 형식으로 출력 가능.

dk = 디스크 메모리 전체 현황 확인

-BM = 메가바이트 수.

-BG = 기가바이트 수.

grep -LVM을 하면 내가 사용하는 lvm으로 나눈 파티션들을 가져오고 거기서 2번째 행이 각 파티션의 전체 용량(GB), 3번째 행이 사용중인 용량(MB)이 뜬다. 두번째 행($2)을 sum += $2 로 총합을 구해주고, 세번째 행($3)을 sum += $3으로 총합을 구해준다.

mpstat으로 cpu잉여량을 구해서 반대로 바꿔 현재 사용량을 출력.

tail - 1 은 가장 마지막 행을 가져온다. 거기서 $NF를 하면 마지막 행을 가장 밑(마지막) 필드를 가리킴.

last reboot 명령어로 리부팅한 시간을 검색이 가능하지만 과제 형식과는 잘 맞지않아 who -b로 출력.

who = 현재 시스템에 로그인한 사용자의 정보를 보여준다. 

who - b를 하면 가장 마지막 부팅 시간만 가져오기 가능.

$3, $4만 출력시킴.

LVM사용 여부는 lsblk를 사용, 나온 출력에서 LVM 문자열이 있는지 grep으로 탐색후, wc -l로 줄 갯수 값을 받아 있으면 1이상, 없으면 0으로 결과값이 나옴.

이 결과값을 if($1>0)이라는 조건을 통해 참(”yes”) 거짓(”no”)을 판별.

# 여담(파이프 관련)

파이프(**`|`**)는 앞의 명령어의 출력을 뒤의 명령어로 전달한다. 

따라서 뒤의 명령어가 먼저 실행되고 앞의 명령어가 실행되는 경우, 뒤의 명령어는 앞의 명령어의 출력을 받을 수 없다.

파이프는 일반적으로 앞에서 뒤로 데이터 흐름을 나타내므로 뒤의 명령어는 먼저 실행되고 출력이 있는 경우에 그 출력을 받아서 처리한다.

**`echo`** 명령어 같은 출력 명령어를 사용하여 명령어의 출력을 포함한 문자열을 만들고자 할 때, 해당 명령어를 서브셸로 감싸서 명령어의 실행 결과를 가져올 수 있다. 이를 명령어 치환(command substitution)이라고 한다.

방법은 두가지.

1. **백틱(backtick) 사용**

```bash
echo "현재 디렉토리는: `pwd`"
```

2.**`$()` 사용**

```bash
echo "현재 디렉토리는: $(pwd)"
```
