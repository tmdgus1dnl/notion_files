# 🌉 Multi-Robot Bridge Daemon (`bridge_app`)

이 어플리케이션은 **여러 대의 로봇(NVIDIA Jetson 탑재)과 외부 모니터링 시스템(Qt App 및 원격 PC)을 연결하는 고성능 통신 중계(Bridge) 데몬**입니다. 주로 **Raspberry Pi 5 (RPi5)** 환경에서 구동되며, 로봇이 전송하는 막대한 양의 센서 데이터(카메라, 라이다)를 실시간으로 병합하여 지연 없이 UI 어플리케이션에 전달하고, 반대로 UI의 조이스틱 제어 명령을 로봇으로 전달하는 핵심 브레인 역할을 수행합니다.

---

## 🚀 주요 기능 (Features)

1. **다중 로봇 동시 지원 (Multi-Robot Support)**
    - 단일 UDP 소켓으로 최대 10대의 로봇 통신을 동시에 수용 (`MAX_ROBOTS = 10`)
    - 로봇 식별자(`robot_id`)를 기반으로 패킷을 자동으로 분류하여 로봇별로 데이터 격리 보장
2. **초고속 UDP 단편화 및 조립 (Fragmentation & Reassembly)**
    - 대용량 데이터(JPEG 이미지, LiDAR 포인트 클라우드)가 UDP MTU(1400 bytes)를 초과할 경우, 이를 분할 수신한 뒤 자체 조립 슬롯(Reassembly Slot)에서 완벽한 1프레임으로 복원.
    - 네트워크 유실 시 100ms 타임아웃 처리를 통해 불완전한 폐기 데이터를 자동 수거(Garbage Collection).
3. **고성능 IPC 및 Lock-Free 메모리 공유 (Shared Memory)**
    - Qt 기반 UI 앱과의 통신 프로세스 오버헤드를 없애기 위해 **POSIX 공유 메모리 (`/robot_bridge_N`)** 사용
    - **라이다(LiDAR)**: Double Buffer (2중 버퍼)
    - **카메라 이미지(Image)**: Triple Buffer (3중 버퍼)를 적용하여 화면 깨짐(Tearing)이나 Lock 경합 없이 극한의 처리 속도 제공
    - 프레임 조립 완료 즉시 POSIX Semaphore(`img_sem`, `lidar_sem`)를 통해 UI를 즉각 깨워(Notify) 디스플레이 반응 속도 최적화.
4. **하이브리드 커맨드 인터페이스**
    - **Qt UI 전용**: Unix Domain Socket (`/tmp/bridge_cmd.sock`)을 통해 로컬 앱의 초저지연 조종 커맨드 수용
    - **원격 PC 전용**: UDP(Port 9002)를 개방하여 별도의 데스크탑/노트북에서도 로봇 원격 관제 및 심박수(Heartbeat) 파악 가능

## 🏗️ 시스템 아키텍처 및 스레드 설계

본 데몬은 커널 자원 병목을 피하고 패킷 드롭을 최소화하기 위해 **철저한 역할 분담형 멀티 스레딩 아키텍처**로 설계되었습니다.

| 스레드 이름 | 갯수 | 역할 및 설명 (실시간성) |
| --- | --- | --- |
| **`main (Watchdog)`** | 1개 | 전역 브릿지 컨텍스트(SHM 등)를 초기화하고, 워크 스레드들을 스폰시킵니다. 이후 백그라운드 루프를 돌며 Odom 패킷의 갱신 여부를 판독해 로봇 네트워크 끊김(Disconnect)을 감지합니다. |
| **`jetson_rx`** | 1개 | **(★★★★ 최고 우선순위, 특정 CPU Core 고정)**
UDP 포트 `9000`에서 로봇들의 모든 데이터를 가장 먼저 수직 수신합니다. Odom은 공유메모리에 직행시키고, 무거운 이미지/LiDAR 패킷은 `FragQueue`에 신속히 적재하여 커널 UDP 버퍼가 터지는 것을 막습니다. |
| **`reassembly_shm`** | N개 | **(로봇 갯수만큼 동적 생성)**
`FragQueue`에서 패킷 조각을 꺼내와 타입별로 조합합니다. 한 장의 프레임이 완성되면 Triple/Double Buffer의 최신 슬롯에 덮어씌운 뒤, Semaphore를 통해 렌더러에 알림(Post)을 보냅니다. |
| **`jetson_tx`** | 1개 | Unix 스레드 소켓(`BRIDGE_CMD_SOCK`) 서버. Qt App이 보내는 조종 명령을 즉시 낚아채어 IP 라우팅 테이블(자동 학습됨)을 참조한 뒤, 해당 로봇의 UDP 포트 `9001`로 발사합니다. |
| **`protocol_timer`** | 1개 | 1Hz 주기로 모든 등록된 Jetson 노드를 향해 Heartbeat 패킷을 쏩니다. 이는 로봇(Jetson) 측에서 연결 유실 시 안전 정지(E-Stop) 기능을 수행할 수 있게 하는 생명줄 역할을 합니다. |
| **`pc_link`** | 1개 | 외부 PC 시스템 관제용 스레드. `9002` 포트를 통해 외부에서 날아오는 커맨드 패킷 처리 및 데몬의 메타 상태 통계자료 전송. |

## 📡 프로토콜 스펙 명세 (Protocol Specification)

모든 통신 패킷의 앞단에는 무조건 24Bytes 크기의 공통 헤더(`PktHeader`)가 붙어있습니다.

### Header 포맷 (`PktHeader`)

- `type`: 패킷 종류 (0x01: IMAGE, 0x02: LIDAR, 0x03: ODOM, 0x04: CMD)
- `robot_id`: 송신 대상 로봇의 고유 ID (0 ~ 9)
- `frag_idx` / `frag_total`: (이미지 등) 잘라진 패킷의 순번과 총 개수
- `frame_id`: 시퀀스 번호 (중복 수신 방지용)
- `payload_offset`: 전체 버퍼 중 현재 조각이 부착되어야 하는 메모리 시작점(offset)

### 네트워크 포트 (Ports)

- **`Port 9000 (UDP Rx)`**: RPi5가 데이터를 수신하는 주 포트.
- **`Port 9001 (UDP Tx)`**: Jetson 제어 커맨드가 도착하는 대상 포트.
- **`Port 9002 (UDP PC)`**: 외부 시스템 관제용 포트.

## 🔧 빌드 및 실행 (Build & Run)

### 의존성

- C11 기반 GCC 컴파일러
- POSIX Threads (`pthread`), Real-Time Library (`lrt`)
- 리눅스 기반 커널(Linux Kernel)

### 빌드 명령어

```bash
# 기본 릴리즈 빌드
make

# 위생기(Sanitizer)를 포함한 디버깅용 빌드
make debug

# 바이너리 클리어
make clean
```

### 실행 방법

```bash
# 기본 실행 (로봇 1대 기준)
./bridge_daemon

# 다중 로봇 실행 (예: 3대)
./bridge_daemon 3
```
