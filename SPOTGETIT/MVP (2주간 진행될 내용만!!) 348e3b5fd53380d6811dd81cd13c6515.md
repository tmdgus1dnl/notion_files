# MVP (2주간 진행될 내용만!!)

- 본 프로젝트는 하드웨어 및 보행 RL 트랙, 관제 트랙, 디지털트윈 트랙의 3개 트랙을 병렬로 진행하며, 
MVP는 세 트랙이 하나의 공통 인터페이스(ROS2 토픽 스키마) 위에서 만나도록 뼈대를 세우는 것을 목표로 합니다.

## 하드웨어 / 보행 RL 트랙

담당자 : @제영호  @구진영 

- Spot Micro 조립, STM32 펌웨어, Jetson Orin Nano 셋업을 2주 시점까지 완성된 부분만 실물 로봇으로 데모에 투입합니다.
- RL 트랙에서는 로봇 보행 RL 학습을 계속 진행하되, MVP 에서는 기본 보행 컨트롤러나 수동 조종으로 대체하고, 학습이 완성되는 대로 교체합니다.

## 관제 트랙

담당자 : @승현 한 @경근 배 

- 웹 기반 관제 UI에 전술 맵, 로봇별 색상 경로 실선, 이벤트 패널, 로봇 상세 상태 패널을 구현하여 3대(실물 1 + 가상 2)가 한 화면에서 실시간 모니터링되게 합니다.
- 각 디바이스간 통신은 커스텀 udp프로토콜을 사용합니다.

## Perception/Localization/Digital Twin 트랙

담당자 : @IG_Bang @김병우 남자 

[Perception]

- Camera : 카메라를 통해 Spot의 주변 환경 인식 및 실종자 탐지
- LiDAR : LiDAR를 통해 주변 환경 장애물과의 거리&각도 추출

[Localization]

- odom + IMU + LiDAR 데이터를 활용한 현재 Spot의 위치 계산 → odom, IMU 데이터 필수
- 공통좌표계 기반 현재 Spot 위치 Localization
- global path, local path 생성

[임무 할당 알고리즘]

- Localization에서 생성된 global path set 중 로봇 개수 만큼 임무 할당

[Digital Twin]

- 실제 로봇과 Iassc Sim 시뮬레이터 엔티티 연결
- 가상 로봇 2대와 실제 로봇 디지털트윈 환경에 로드
- 디지털트윈 트랙에서는 Isaac Sim(Zone A/B/C + 장애물)와 가상 로봇 2대(sim_01, sim_02)를 구축하고, 실물·가상 모두 동일 스키마로 /odom, /robot_status, /detection 토픽을 발행하도록 표준화합니다.
