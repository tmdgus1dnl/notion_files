## **1. 공고 요약**

### **포지션 소개**

휴머노이드 로봇의 실시간 제어 시스템 및 소프트웨어 플랫폼을 개발하는 직무. 운영체제 환경 구성, 디바이스 드라이버, 실시간 제어 프레임워크 등 로봇 동작의 핵심 기반이 되는 소프트웨어를 설계 및 개발하며, 하드웨어 설계 조직 및 AI 연구 조직과 긴밀히 협업.

### **수행업무**

1. 휴머노이드 로봇의 실시간 제어 프레임워크를 설계하고 개발
2. 모터, 센서 등 로봇 하드웨어 제어를 위한 디바이스 드라이버 및 하드웨어 추상화 계층 개발
3. 로봇 제어용 운영체제(Linux 기반 실시간 OS 등) 환경을 구성하고 시스템 성능을 최적화
4. 유관 부서와 협업하여 보행·조작 등 AI 기능과 제어 시스템 간 연동 미들웨어를 개발

### **자격요건**

- 컴퓨터, 전기·전자, 기계, 로봇공학 등 관련 전공
- 운영체제 기본 개념에 대한 이해도를 보유하신 분
- 요구사항을 분석하여 소프트웨어를 구조적으로 설계 및 구현하는 역량을 보유하신 분
- 다양한 분야의 엔지니어와 적극적으로 소통하며 협업할 수 있는 역량을 보유하신 분

### **우대사항**

- C/C++ 기반 시스템 프로그래밍 역량을 보유하신 분
- Linux 기반 임베디드 시스템 개발 경험을 보유하신 분 (Kernel, Device Driver, BSP 등)
- CAN, EtherCAT, UDP 등 하드웨어 통신 프로토콜 활용 경험을 보유하신 분
- ROS2 기반 로보틱스 프로젝트 수행 경험을 보유하신 분
- Git 기반 협업 및 CI/CD 환경에서의 개발 경험을 보유하신 분

## **2. 전체 학습 로드맵**

임베디드 레시피 강좌는 `[레시피]`, 문c 블로그는 `[문c]`, 별도 학습은 `[외부]`로 표기.

### **Phase 1. 하드웨어와 시스템의 기초 체력 만들기**

`[레시피]` **1장 Hardware 꼴라쥬** 회로도 읽기, 신호와 주파수, 디지털/아날로그 신호, 저항/인덕터/캐패시터, 트랜지스터, Pull-up/Pull-down, 논리회로, Register, Clock, Bus Transfer, Timing/Spec 읽기, Memory, CPU 동작 원리까지.

`[레시피]` **2장 Microprocessor 아뜰리에** ARM 아키텍처, 동작 Mode, Register와 Context, Exception과 Mode, Interrupt 처리, ARM SoC 구조, AMBA 버스 규격까지.

### **Phase 2. 소프트웨어 빌드 체계와 C 심화**

`[레시피]` **3장 Software 데꾸바쮸** 컴파일 과정, 전처리, 어셈블리 변환, 라이브러리, 컴파일러 옵션, 변수의 scope와 Memory Map, ELF 포맷, Linker, Scatter Loading, Map file 분석, Makefile까지.

`[레시피]` **5장 Software 비네팅** AAPCS, 포인터와 배열, struct/typedef/PACKED, Stack과 Heap 깊이 이해, 함수 호출 시 스택 동작, Stack Size 산정, 함수 포인터, Linked List와 Queue까지.

**[외부] C++ 시스템 프로그래밍** 클래스 설계, 상속과 다형성, 스마트 포인터, RAII 패턴, 템플릿, STL 컨테이너, move semantics. 임베디드/시스템 관점에서 메모리 효율과 실시간 성능을 고려하는 방식으로 학습.

### **Phase 3. RTOS 원리 — 이 직무의 핵심**

`[레시피]`**6장 RTOS 팩토리 (전체를 가장 정성 들여 학습)** RTOS와 Kernel, 무한 Loop 구조, Task 구조/Signal/상태, 선점형 Multitasking, Context Switching/TCB, Scheduler 구현, ISR 구현, Clock tick ISR, Critical Section/Mutex/Semaphore, Interrupt vs Polling, Queue와 Inter Task Communication, DPC/APC/Bottom Half, Watchdog, Kernel 진입과 Porting까지.

> 공고의 4개 수행업무 전부와 교차하는 가장 핵심적인 장. 두세 번 반복 학습 권장.
> 

### **Phase 4. 디바이스 제어와 시스템 아키텍처**

`[레시피]`**7장 Device Control and System Architecture** Device control 원리, Register Setting/Memory Mapped IO, Memory Device 제어, Bus Sizer/Memory Controller, Shadow와 Bit operation, MACRO 테크닉, Synchronous/Asynchronous, Wait State, PLL, GPIO와 GPIO ISR, DMA, Cache, MMU, JTAG까지.

`[레시피]`**4장 ARM 미장센 (핵심 선별)** Assembly 기초와 Reverse Engineering, Exception Vector Table과 Handler 구현, Bootloader와 Memory Budget, Reset Handler에서 main까지.

### **Phase 5. 디버깅 역량**

`[레시피]`**8장 How to debug (책에서만 열람 가능)** Data Abort, Stack Corruption, Prefetch Abort, Watchdog Reset, 다양한 Exception과 Memory 불량 증상, JTAG 디버깅 등.

### **Phase 6. Linux 커널과 디바이스 드라이버**

**[외부] Linux 커널 기초** 프로세스 관리, 스케줄링, 메모리 관리(가상 메모리, 페이지 테이블), 파일 시스템, 시스템 콜 인터페이스.

**[문c] 8장 디바이스 드라이버** ← 최우선 Device & Driver 기본 구조(Bus & Class, Platform Device), Pin Control Subsystem, GPIO Subsystem, I2C/SPI/PCI Subsystem.

**[문c] 7장 스케줄러, Mutex, RCU** CFS/RT/Deadline Scheduler, Preemption & Context Switch, PELT, Load Balance, Mutex, RCU.

**[문c] 6장 인터럽트/타이머** GIC v3 Driver, Top-Half & Bottom-Half, Softirq, Workqueue, HRTimer, Clock Source, Tick Device.

**[문c] 5장 중 DMA/CMA** CMA(Contiguous Memory Allocator), DMA 1~6, Zoned Allocator, Slub 등.

**[외부] Linux Device Driver 개발** 캐릭터 디바이스 드라이버, 커널 모듈, Platform Driver, Device Tree, I2C/SPI/UART 서브시스템, interrupt handling, DMA 매핑.

**[외부] BSP(Board Support Package)** 커널 컨피그, 부트로더(U-Boot) 설정, 크로스 컴파일, rootfs 구성, Yocto/Buildroot.

**[외부] Linux 실시간 확장** PREEMPT_RT 패치, RT 스케줄링 정책(SCHED_FIFO, SCHED_RR), 레이턴시 측정과 최적화, cyclictest.

### **Phase 7. 산업용 통신 프로토콜**

**[외부] CAN(Controller Area Network)** CAN 2.0A/B 프레임 구조, 중재 메커니즘, 에러 처리, Linux SocketCAN 인터페이스.

**[외부] EtherCAT** 실시간 산업용 이더넷, 마스터/슬레이브 구조, 분산 클록, SOEM 오픈소스 스택.

**[외부] UDP/이더넷 통신** 소켓 프로그래밍, 비연결형 실시간 데이터 전송, 원격 관제 패턴.

### **Phase 8. ROS2와 로보틱스 미들웨어**

**[외부] ROS2 기초** 노드, 토픽, 서비스, 액션, 파라미터, 퍼블리셔/서브스크라이버 패턴(C++).

**[외부] ROS2 심화** DDS 미들웨어, QoS 설정, 실시간 노드(real-time executor), ros2_control 프레임워크, lifecycle node.

### **Phase 9. 개발 인프라와 협업 도구**

**[외부] Git** 브랜치 전략(Git Flow), 리베이스, 충돌 해결, 코드 리뷰(PR/MR).

**[외부] CI/CD** Jenkins, GitHub Actions, GitLab CI 등으로 빌드 자동화, 유닛 테스트 자동 실행, 크로스 컴파일 파이프라인.

## **4. 문c 블로그 — 이 직무 기준 학습 우선순위**

문c 블로그는 ARM64 리눅스 커널 소스를 함수 단위로 분석하는 고급 자료. 임베디드 레시피 Phase 1~5를 소화한 후, 커널 이론서를 한 권 읽고 나서 진입하는 것을 권장.

### **선행 조건 (블로그 자체 권장)**

- C 프로그래밍 중급
- Linux bash shell, GCC, Git, Vim
- 간단한 커널 이론서
- ARM 및 ARM64 아키텍처 이론서

### **이 직무 기준 우선순위**

**1순위 — 8장 디바이스 드라이버** → 공고 수행업무 ② "디바이스 드라이버 및 하드웨어 추상화 계층 개발"에 직결

- Device & Driver 1~3 (Basic, Bus & Class, Platform Device)
- GPIO Subsystem 1~4
- I2C Subsystem 1~4
- SPI Subsystem 1~2
- Pin Control Subsystem 1~2

**2순위 — 7장 스케줄러/Mutex/RCU** → 공고 수행업무 ③ "Linux 기반 실시간 OS 환경 구성 및 시스템 성능 최적화"에 대응

- Scheduler 1, 5~7 (Basic, Core, CFS, Preemption & Context Switch)
- Scheduler 9 (RT Scheduler) ← 실시간 제어에 직결
- Scheduler 10 (Deadline Scheduler) ← 실시간 제어에 직결
- Mutex for linux kernel space
- Priority Inversion & Priority Inheritance

**3순위 — 6장 인터럽트/타이머** → 공고 수행업무 ① "실시간 제어 프레임워크"의 리눅스 계층 이해

- Interrupts 1~5 (Controller, irq chip, irq domain, Top/Bottom Half, Softirq)
- Interrupts 7~8 (Workqueue)

## **6. 시간이 없을 때 — 최소 핵심 학습 경로**

임베디드 레시피 내에서 절대 빠지면 안 되는 코어 강좌만 추리면:

```
6장 전체
→ 7장 ⓐⓑⓚⓛⓜⓝ
→ 5장 ⓒⓓⓔⓕⓗⓘⓙ
→ 3장 ⓘⓚⓛⓜⓝⓞⓟ
→ 2장 ⓓⓔⓕⓗⓘⓙ
→ 1장 ⓚⓛⓜⓝⓠⓡ
→ 4장 ⓖⓙⓚ
```

이후 문c 블로그에서:

```
8장 디바이스 드라이버 > 7장 스케줄러/Mutex > 6장 인터럽트/타이머 > 5장 DMA/CMA
```
