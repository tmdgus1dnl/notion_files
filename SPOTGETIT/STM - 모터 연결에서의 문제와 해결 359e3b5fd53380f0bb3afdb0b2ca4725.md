# STM  - 모터 연결에서의 문제와 해결

# SpotMicro Phase 1 — 펌웨어 통신 검증 디버깅 로그

STM32F446RE ↔ STS3215 12 서보 (4 UART HDSEL 버스) 통신 검증 과정에서 만난 문제들.

---

## 1. CubeMX 자동 생성 파일과 사용자 코드 충돌

### 원인

CubeMX가 `usart.c`, `syscalls.c` 등에 `huart1`, `_write`, `Error_Handler` 등을 자동 생성. 사용자 `main.c`에도 같은 정의가 있어서 **링커가 multiple definition 에러** 발생.

### 해결

- `Core/Src/usart.c`, `Core/Inc/usart.h` 삭제
- `Core/Src/syscalls.c`에서 `_write` 함수만 삭제
- `main.c`의 `Error_Handler`를 `static` 떼서 외부 노출

---

## 2. DMA IRQ 핸들러가 삭제된 DMA 핸들 참조

### 원인

`stm32f4xx_it.c`의 `DMA1_Stream0_IRQHandler` 등이 `hdma_uart5_rx` 같은 핸들을 참조. 그런데 이 핸들들의 정의는 삭제한 `usart.c`에 있었어서 **link 에러**.

### 해결

`stm32f4xx_it.c`에서 UART/USART DMA IRQ 핸들러 8개의 내용 삭제 (또는 `HAL_DMA_IRQHandler(...)` 줄 주석).
**SPI DMA 핸들러 (`DMA2_Stream0`, `DMA2_Stream3`)는 유지** — `spi.c`에 핸들 정의 있음.

## 3. USART3 핀 불일치 (CubeMX 설정 vs 코드)

### 원인

`main.h`는 `Rear_Right_Leg_Pin = GPIO_PIN_10, GPIOB` (PB10)로 정의됐지만, `main.c`의 `MX_USART3_HDSEL_Init`은 PC10으로 설정.

### 해결

`main.c`에서 USART3 init을 PB10으로 수정:

```c
gp.Pin = GPIO_PIN_10;
gp.Alternate = GPIO_AF7_USART3;
HAL_GPIO_Init(GPIOB, &gp);   // GPIOB
```

## 4. PING 응답이 1바이트 시프트 (HDSEL Echo/Sync)

### 증상

12 서보 PING 시도 시 6바이트는 받는데 형식이 깨짐:

```
expected: FF FF 01 02 00 FC
received: FA FF FF 01 02 00   ← 1바이트 밀림
```

### 원인

HDSEL 송신 직후 라인이 idle (HIGH)로 안정되기 전에 RX 시작 → 첫 바이트 자리에 잡 byte 한 개 들어옴 → 실제 응답이 1바이트 시프트.

### 해결

**7바이트 RX + 헤더 sync 검색**:

- 6바이트가 아니라 7바이트 받기
- 받은 데이터에서 `FF FF` 헤더 위치 찾기 (offset 0 또는 1)
- 그 위치부터 ID/길이/checksum 검증

```c
HAL_UART_Receive(huart, raw, 7, timeout);
for (int off = 0; off + 5 < received; off++) {
    if (raw[off] == 0xFF && raw[off+1] == 0xFF && raw[off+2] == id) {
        // 응답 발견, 파싱
    }
}
```

## 5. Nucleo PA9가 USB OTG VBUS 회로와 충돌

### 증상

USART3, USART6, UART5는 응답 (sync@1) 받는데 **USART1 (PA9)만 0바이트** 응답.

### 원인

Nucleo F446RE 보드에서 PA9가 **USB OTG VBUS 감지 회로**에 묶여있음. VBUS 쪽 큰 커패시터 + 풀다운이 Open-Drain TX 신호를 흡수해서 1Mbps 통신 불가.

### 해결

**FR 다리를 USART1(PA9)에서 UART4(PA0)로 이동**.

- CubeMX에서 USART1 비활성화, UART4 활성화 (Mode: Single Wire Half-Duplex, AF Open Drain)
- 노란색 점퍼를 PA9 → PA0로 물리 이동 (Arduino A0 헤더)
- `main.c`에서 `huart1` → `huart4`, `MX_USART1_HDSEL_Init` → `MX_UART4_HDSEL_Init` (PA0, AF8) 교체

## 최종 결과

```
=== Cycle N ===
  [OK] FR hip   ID= 4  (sync@1)
  [OK] FR thigh ID= 5  (sync@1)
  ... (12개 모두)
  Summary: 12 / 12 alive
```

4 UART HDSEL × 3 서보 = 12 서보 100% 통신 안정화.
