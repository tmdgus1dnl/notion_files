# 제영호 클로드의 분석 결과…

# SSAFY 임베디드 트랙 6인 / 7주 프로젝트 적합도 순위

6명의 브레인스토밍 문서를 모두 확인했습니다. 제영호, 구진영, 한승현, 방인규, 배경근, 김병우 총 6명의 아이디어를 **6명 팀 구성 / 7주 기간 / 임베디드 트랙**이라는 조건에 맞춰 평가했습니다.

---

## 🥇 1등: 제영호 — RTScope (FreeRTOS 실시간 트레이서)

**가장 적합한 이유:**

**스코프가 정확하다.** 순수 펌웨어 + 호스트 도구라는 깔끔한 구조로, 하드웨어 구매·조립·회로 설계가 사실상 필요 없습니다. STM32 보드 하나면 됩니다. 하드웨어 리스크가 거의 제로이기 때문에 7주 안에 완성 가능성이 가장 높습니다.

**6명 역할 분배가 이미 완벽하게 설계되어 있습니다.** Transport, Ring Buffer, Trace Engine, Timestamp+ISR, Metrics, 통합+Python 디코더 — 각자 독립적인 모듈을 맡아 병렬 개발이 가능하고, W1에 전원 공통 과제 후 W2부터 분리하는 현실적 플랜입니다.

**임베디드 깊이가 최상입니다.** HAL 미사용, CMSIS 레지스터 직접 제어, DMA, FreeRTOS 스케줄러 내부 hook — 면접에서 "로우레벨을 정말 이해한다"는 걸 증명하기에 최적입니다.

**오픈소스로서 실제 가치가 있습니다.** Segger SystemView, Tracealyzer 같은 상용 도구의 무료 대안이라는 포지셔닝이 명확하고, GitHub 스타를 받을 수 있는 현실적인 프로젝트입니다.

**유일한 약점:** 눈에 보이는 데모(로봇이 움직인다 등)가 없어서 발표 시 시각적 임팩트가 약할 수 있지만, 터미널 로그 시각화로 충분히 커버 가능합니다.

---

## 🥈 2등: 구진영 — ROS2-FreeRTOS 범용 모바일 로봇 하부 제어 패키지

**적합한 이유:**

**기획 완성도가 매우 높습니다.** 시스템 아키텍처, 기술 스택, 7주 마일스톤, 6명 역할(Team A/B/C × 2명씩)이 구체적으로 설계되어 있습니다. 한승현의 아이디어와 방향이 비슷하지만, 구진영 버전이 **7주에 맞춰 설계**되어 있고 한승현 버전은 9주입니다.

**기술 범위가 넓으면서도 계층적으로 잘 분리됩니다.** HAL/드라이버 → FreeRTOS/micro-ROS → ROS2 Host 진단으로 레이어가 나뉘어 팀원별 학습 부담이 분산됩니다.

**포트폴리오 가치가 높습니다.** ROS2 생태계 기여, C++ HAL 설계, 실시간 성능 벤치마킹 등 면접에서 어필할 포인트가 다양합니다.

**주의할 점:** 하드웨어(모터, 엔코더, 샤시 등)가 필요하므로 부품 수급·조립에 1~2주가 소모될 리스크가 있습니다. 또한 micro-ROS 포팅 자체가 난이도가 높아 초반에 막히면 전체 일정이 밀릴 수 있습니다. 하드웨어 의존성이 RTScope 대비 높다는 점이 2등인 이유입니다.

## 🥉 3등: 김병우 — 다중 노드 분산 제어 시스템

**적합한 이유:**

**임베디드의 핵심인 "다중 MCU 간 통신"을 다룹니다.** Sensor Node / Master Control Node / Actuator Node로 나뉜 구조는 자동차 ECU 네트워크와 유사하여 산업 현장 관련성이 높습니다.

**데모가 직관적입니다.** 센서가 위험 감지 → Master가 판단 → Actuator가 모터 정지/경보 출력이라는 흐름이 눈에 보여서 발표 임팩트가 좋습니다.

**보드 3개 + CAN/UART 통신이라는 구조가 6명에게 자연스럽게 분배 가능합니다** (노드당 2명).

**3등인 이유:** 기획서에 주간 일정, 구체적 역할 분담, 기술 스택 정리가 없습니다. 아이디어 자체는 좋지만, 현재 상태로는 "어디까지 할 것인가"의 스코프가 불명확합니다. CAN 통신을 쓸 경우 CAN 트랜시버 모듈이 필요하고, FSM 설계·구현 범위가 모호합니다. 7주 안에 완성하려면 기획을 훨씬 구체화해야 합니다.

---

## 나머지 아이디어 간략 평가

**한승현(ROS2 범용 MCU 펌웨어):** 구진영과 거의 동일한 방향이지만 **9주 기획**이라 7주에 안 맞고, 젯슨 Orin Nano + Nav2 자율주행까지 포함하여 스코프가 너무 넓습니다. 구진영 버전이 더 현실적입니다.

**방인규:** 아이디어가 5~6개로 분산되어 있고, 개별 아이디어(하드폴트 분석, FOTA 부트로더 등)는 기술적으로 훌륭하지만 대부분 **1~2명 규모**의 프로젝트입니다. 6명이 7주간 할 만한 볼륨으로 확장하기 어렵습니다.

**배경근:** 블록 코딩, 밸런싱 로봇, 와이파이 CSI — 아이디어 제목만 있고 상세 내용이 없어 평가가 어렵습니다. 밸런싱 로봇은 재미있지만 6명이 7주간 하기엔 스코프가 작고, 와이파이 CSI는 임베디드보다 신호처리/ML 비중이 높아 트랙 적합도가 낮습니다.
