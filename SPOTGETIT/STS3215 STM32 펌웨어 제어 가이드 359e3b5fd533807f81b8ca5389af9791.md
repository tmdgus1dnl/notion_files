# STS3215 STM32 펌웨어 제어 가이드

> 4족 보행 로봇 (STM32F446RE + 12× STS3215) 펌웨어 설계 문서
기반 자료: Feetech Servo Tool 분석 문서
통신 인터페이스: USART HDSEL (Half-Duplex) + 1Mbps + DMA
> 

---

## 1. 시스템 개요

### 1.1 하드웨어 구성

```
Jetson Orin ──SPI(slave)── STM32F446RE ──┬── USART1(HDSEL) ── FR 다리 (3 모터)
                                          ├── USART3(HDSEL) ── RR 다리 (3 모터)
                                          ├── USART6(HDSEL) ── FL 다리 (3 모터)
                                          ├── UART5(HDSEL)  ── RL 다리 (3 모터)
                                          └── I2C1 ── BNO055 IMU
```

### 1.2 핀 매핑

| 다리 | UART | TX 핀 | 모터 ID |
| --- | --- | --- | --- |
| FR (Front Right) | USART1 | PA9 | 1, 2, 3 (hip, thigh, knee) |
| FL (Front Left) | USART6 | PC6 | 4, 5, 6 |
| RR (Rear Right) | USART3 | PB10 | 7, 8, 9 |
| RL (Rear Left) | UART5 | PC12 | 10, 11, 12 |

### 1.3 통신 사양

- **보드레이트**: 1 Mbps (1 Mbps 권장 — 12 모터 50–100 Hz 제어용)
- **모드**: Half-Duplex (HDSEL) — 1선 양방향 통신
- **외부 풀업**: 1 kΩ → +5V (이미 결선됨)
- **공통 GND**: 12V 모터 전원 GND ↔ STM32 GND 필수

## 2. STS3215 프로토콜 패킷 구조

### 2.1 패킷 포맷

```
┌──────┬──────┬─────┬────────┬───────────┬────────────┬──────────┐
│ 0xFF │ 0xFF │ ID  │ Length │ Instruct  │ Parameters │ Checksum │
└──────┴──────┴─────┴────────┴───────────┴────────────┴──────────┘
   1B     1B    1B     1B         1B        N bytes        1B

Length = N + 2  (Instruction + Params + Checksum)
Checksum = ~(ID + Length + Instruct + Param1 + ... + ParamN) & 0xFF
```

### 2.2 Instruction 종류

| Instruction | 값 | 용도 |
| --- | --- | --- |
| PING | 0x01 | 응답 확인 (서보 살아있나?) |
| READ | 0x02 | 레지스터 읽기 |
| WRITE | 0x03 | 레지스터 쓰기 |
| REG_WRITE | 0x04 | 명령 예약 (ACTION 대기) |
| ACTION | 0x05 | REG_WRITE 일괄 실행 |
| RESET | 0x06 | 공장 초기화 |
| SYNC_WRITE | 0x83 | **다중 서보 동기 쓰기 (보행 핵심!)** |

### 2.3 핵심 레지스터 주소 (STS 시리즈)

### EEPROM (셋업 시 1회만 쓰기)

| Addr | 이름 | Size | 설명 |
| --- | --- | --- | --- |
| 5 | ID | 1 | 서보 식별자 (1–253) |
| 6 | Baud Rate | 1 | 0=1M, 4=115200 |
| 9 | Min Position Limit | 2 | 위치 하한 (step) |
| 11 | Max Position Limit | 2 | 위치 상한 (step) |
| 16 | **Max Torque Limit** | 2 | 0.1% 단위, 1000=100% |
| 21 | Position P Gain | 1 | PID P |
| 22 | Position D Gain | 1 | PID D |
| 23 | Position I Gain | 1 | PID I |
| 31 | Position Offset | 2 | 영점 보정 (-2047~2047) |
| 33 | Work Mode | 1 | 0=위치, 1=휠 |
| 34 | Protect Torque | 1 | 보호 발동 후 유지 토크 (%) |
| 35 | Overload Protection Time | 1 | 10ms 단위 |
| 36 | Overload Torque | 1 | 과부하 임계치 (%) |
| 55 | Lock | 1 | EEPROM 쓰기 잠금 |

### SRAM (런타임 제어)

| Addr | 이름 | Size | 설명 |
| --- | --- | --- | --- |
| 40 | **Torque Enable** | 1 | 0=OFF, 1=ON, 128=영점 보정 |
| 41 | Goal Acceleration | 1 | 100 step/s² |
| 42 | **Goal Position** | 2 | step (-32766~32766) |
| 46 | Goal Velocity | 2 | step/s (-1000~1000) |
| 48 | **Torque Limit** | 2 | 0.1% 단위, 런타임 동적 변경 |
| 55 | Lock | 1 | EEPROM 잠금 |
| 56 | Current Position | 2 | R/O, 현재 위치 |
| 58 | Instantaneous Velocity | 2 | R/O, step/s |
| 60 | **Current PWM (Torque)** | 2 | R/O, 10-bit + 부호 |
| 62 | Voltage | 1 | R/O, 0.1V |
| 63 | Temperature | 1 | R/O, °C |
| 65 | **Hardware Error** | 1 | R/O, 비트 플래그 |
| 66 | Moving Status | 1 | R/O, 0=정지, 1=이동 중 |

## 3. STM32 HDSEL UART 통신 패턴

### 3.1 동작 원리

HDSEL 모드는 **TX 핀 1개**로 송수신을 다 함. 하드웨어가 자동으로 방향 전환.

```c
// 송신: TX 모드 → 데이터 전송 → TC(Transmission Complete) 인터럽트
// 자동 전환: TC 후 RX 모드
// 수신: 서보 응답 받기 → DMA로 자동 수신
```

### 3.2 송수신 함수 (DMA + 인터럽트)

```c
typedef enum {
    BUS_IDLE,
    BUS_TX,
    BUS_RX,
    BUS_TIMEOUT
} BusState;

typedef struct {
    UART_HandleTypeDef *huart;
    uint8_t  tx_buf[32];
    uint8_t  rx_buf[32];
    volatile BusState state;
    uint32_t tx_start_tick;
} ServoBus;

ServoBus bus_FR = { .huart = &huart1 };
ServoBus bus_FL = { .huart = &huart6 };
ServoBus bus_RR = { .huart = &huart3 };
ServoBus bus_RL = { .huart = &huart5 };

// 패킷 전송 (DMA)
HAL_StatusTypeDef servo_send(ServoBus *bus, uint8_t *data, uint16_t len) {
    bus->state = BUS_TX;
    bus->tx_start_tick = HAL_GetTick();
    return HAL_UART_Transmit_DMA(bus->huart, data, len);
}

// TX 완료 콜백 (자동 호출)
void HAL_UART_TxCpltCallback(UART_HandleTypeDef *huart) {
    ServoBus *bus = find_bus(huart);
    bus->state = BUS_RX;
    // 응답 수신 시작 (DMA)
    HAL_UART_Receive_DMA(bus->huart, bus->rx_buf, EXPECTED_RX_LEN);
}

// RX 완료 콜백
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    ServoBus *bus = find_bus(huart);
    bus->state = BUS_IDLE;
    parse_response(bus->rx_buf);
}
```

### 3.3 타이밍 주의사항

- **Return Delay Time** (레지스터 7): 서보가 명령 받고 응답까지 대기 시간 (기본 250 → 500µs)
- **응답 길이 미리 알아야 함**: READ 패킷 보내면 받을 길이를 미리 계산
- **타임아웃**: 1ms 정도가 안전 (1 Mbps에서 가장 긴 패킷 ~150µs)
- **Sync Write는 응답 없음**: TX만 하고 끝, RX 안 기다림

## 4. 단계별 구현 계획

각 단계를 통과해야 다음으로 진행. **건너뛰면 디버깅 지옥.**

### 단계 A: 서보 1개 PING 응답 확인 (가장 먼저)

목표: STM32 ↔ 서보 1번 통신 가능 검증

```c
// PING 패킷: 0xFF 0xFF [ID] 0x02 0x01 [checksum]
uint8_t ping_pkt[6] = {
    0xFF, 0xFF,
    0x01,           // ID = 1
    0x02,           // Length
    0x01,           // PING
    0xFB            // ~(1+2+1) & 0xFF = 0xFB
};

// 응답 패킷: 0xFF 0xFF [ID] 0x02 0x00 [checksum]
// 길이 6 바이트, 0x00 = error 없음
```

**성공 기준**: 응답 6 바이트 받기, ID=1, Error=0x00

### 단계 B: 위치 읽기 (Read Position)

```c
// READ Position: 주소 56부터 2 바이트
// 0xFF 0xFF [ID] 0x04 0x02 [Addr] [Read_len] [checksum]
uint8_t read_pos_pkt[8] = {
    0xFF, 0xFF,
    0x01,
    0x04,           // Length
    0x02,           // READ
    56,             // Address: Current Position
    2,              // Read 2 bytes
    /* checksum */
};
// checksum = ~(1 + 4 + 2 + 56 + 2) & 0xFF
```

**성공 기준**: 응답 8바이트, position 값이 ~2048 (중립 상태)

### 단계 C: 작은 위치 명령 테스트

⚠️ **반드시 Torque Limit 낮게 + 작은 변위로**:

```c
// 1. Torque Limit를 250 (25%)로 낮추기
write_word(id=1, addr=48, value=250);

// 2. Torque Enable
write_byte(id=1, addr=40, value=1);

// 3. Goal Position을 중립 +100 (≈9도)
write_word(id=1, addr=42, value=2148);

// 4. 1초 후 위치 읽어서 확인
// 5. 중립으로 복귀
write_word(id=1, addr=42, value=2048);
```

**성공 기준**: 모터가 9도 정도 회전 후 복귀

### 단계 D: 1다리 (3 모터) 통신

같은 USART에 3개 서보 데이지체인. ID 1, 2, 3로 각각 PING.

### 단계 E: 4개 USART 동시 (4다리, 12 모터)

### 단계 F: SyncWrite 구현

12개 모터에 동시에 Goal Position 명령. 보행에 필수.

```c
// SYNC_WRITE 패킷:
// 0xFF 0xFF 0xFE [Length] 0x83 [Start_Addr] [Data_per_servo]
// [ID1][D1_1][D1_2]...[ID2][D2_1][D2_2]... [checksum]
//
// 예: 4 모터 (한 다리)에 Goal Position 동시 쓰기
// Addr=42, Data_per_servo=2 (position만)
```

### 단계 G: 1 kHz 제어 루프 통합

```
TIM 인터럽트 1 kHz
├─ Phase 1 (0–200µs): SyncWrite Goal Position (4 다리 동시)
├─ Phase 2 (200–800µs): 각 다리에서 1 모터씩 Read Position 응답 받기
└─ Phase 3 (800–1000µs): SPI로 Jetson에 피드백 + IMU 처리
```

## 5. 안전 파라미터 (셋업 시 1회만)

처음 12 모터 ID 부여 후, **각 모터에 다음 EEPROM 값 1회 설정**:

```c
void servo_initial_setup(uint8_t id) {
    // EEPROM 잠금 해제
    write_byte(id, 55, 0);

    // 안전 한계
    write_word(id, 16, 800);        // Max Torque Limit = 80%
    write_byte(id, 34, 40);         // Protect Torque = 40 (4%)
    write_byte(id, 35, 50);         // Overload Time = 500ms
    write_byte(id, 36, 70);         // Overload Torque = 70%
    write_byte(id, 13, 70);         // Max Temp = 70°C
    write_byte(id, 15, 90);         // Min Voltage = 9.0V

    // 위치 한계 (관절별로 다름 - URDF 보고 정해)
    write_word(id, 9, hip_min[id]);    // Min Position
    write_word(id, 11, hip_max[id]);   // Max Position

    // 정지마찰 극복
    write_word(id, 24, 30);          // Punch = 3%

    // 영점 보정 (필요시)
    // write_word(id, 31, offset[id]);

    // 잠금
    write_byte(id, 55, 1);
}
```

⚠️ **EEPROM은 쓰기 횟수 제한 있음**. 매 사이클마다 쓰면 수명 소모. **셋업 시 1회만**.

## 6. 라디안 ↔ 모터 step 변환

### 6.1 기본 변환

```c
#define SERVO_TICKS_PER_REV    4096      // 12-bit (정확히 4096이지만 0~4095 사용)
#define SERVO_TICKS_PER_RAD    (4096.0f / (2.0f * M_PI))  // ≈ 651.9
#define SERVO_NEUTRAL          2048

uint16_t rad_to_ticks(float rad_offset_from_neutral) {
    int32_t ticks = SERVO_NEUTRAL + (int32_t)(rad_offset_from_neutral * SERVO_TICKS_PER_RAD);
    if (ticks < 0)    ticks = 0;
    if (ticks > 4095) ticks = 4095;
    return (uint16_t)ticks;
}

float ticks_to_rad(uint16_t ticks) {
    return (ticks - SERVO_NEUTRAL) / SERVO_TICKS_PER_RAD;
}
```

### 6.2 관절별 캘리브레이션 (12개)

```c
typedef struct {
    int8_t  direction;       // +1 또는 -1 (모터 회전 방향이 관절 좌표계와 일치/반대)
    int16_t zero_offset;     // 중립 자세에서 모터 위치 (보통 2048)
    int16_t min_pos;         // 안전 한계 (기구 간섭 방지)
    int16_t max_pos;
} ServoCalib;

// 12개 관절 캘리브레이션 데이터
ServoCalib calib[12] = {
    // ID 1: FR_hip
    { .direction = +1, .zero_offset = 2048, .min_pos = 1024, .max_pos = 3072 },
    // ID 2: FR_thigh
    { .direction = -1, .zero_offset = 2048, .min_pos = 1024, .max_pos = 3072 },
    // ID 3: FR_knee
    { .direction = +1, .zero_offset = 2048, .min_pos = 1024, .max_pos = 3072 },
    // ... 12개 다 채우기 (URDF 보고)
};

uint16_t joint_rad_to_servo(int joint_idx, float joint_rad) {
    float servo_rad = joint_rad * calib[joint_idx].direction;
    int32_t ticks = calib[joint_idx].zero_offset + (int32_t)(servo_rad * SERVO_TICKS_PER_RAD);

    if (ticks < calib[joint_idx].min_pos) ticks = calib[joint_idx].min_pos;
    if (ticks > calib[joint_idx].max_pos) ticks = calib[joint_idx].max_pos;

    return (uint16_t)ticks;
}
```

### 6.3 시뮬 ↔ 실제 일치 검증

RL 정책은 시뮬에서 학습됐을 것. 시뮬의 URDF와 실제 캘리브레이션이 일치해야 함:

| 항목 | 시뮬 | 실제 STM32 |
| --- | --- | --- |
| 관절 0점 | URDF의 origin | calib.zero_offset |
| 회전 부호 | URDF의 axis | calib.direction |
| 가동 범위 | URDF의 limit | calib.min_pos / max_pos + EEPROM Position Limit |

**검증 방법**:

1. 시뮬에서 특정 관절 각도 (예: hip = +0.5 rad) 명령
2. 실제 로봇에 같은 명령
3. 두 자세가 같은지 육안 확인
4. 다르면 direction 부호 또는 zero_offset 조정

## 7. 1 kHz 제어 루프 설계

### 7.1 시간 예산

```
1ms = 1000µs 안에 다음 작업 완료:

SPI 응답 (Jetson에) ────── 50µs
4 USART SyncWrite TX ──── 150µs (병렬, 12 모터 동시)
서보 Return Delay ──────── 500µs (서보 응답 대기)
4 USART Read 응답 RX ──── 200µs
IMU I2C 읽기 ───────────── 100µs (별도 시간 슬롯)
PID/제어 계산 ──────────── 여유

총: ~1ms
```

### 7.2 메인 루프 구조

```c
volatile uint8_t control_tick = 0;

// TIM3 인터럽트 1kHz
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim->Instance == TIM3) {
        control_tick = 1;
    }
}

void main_control_loop(void) {
    while(1) {
        if (!control_tick) continue;
        control_tick = 0;

        // 1. SyncWrite Goal Position (4 다리 동시 시작)
        sync_write_goal_position_all_legs(target_positions);

        // 2. (병렬) 이전 사이클의 응답 처리
        process_servo_feedback();

        // 3. IMU 읽기 (I2C)
        bno055_read_quaternion(&imu_data);

        // 4. SPI 응답 준비 (다음 Jetson 트랜잭션용)
        prepare_spi_tx_buffer();
    }
}
```

### 7.3 비동기 통신 — 가장 중요

**12 서보를 순차 명령하면 절대 안 됨**:

- 1 서보당 패킷 ~100µs × 12 = 1.2ms → 1kHz 못 맞춤

**해결: SyncWrite + 4 UART 병렬**:

- 4 UART 동시에 SyncWrite 송신
- 1 UART당 3 모터 → 패킷 길이 짧음
- 모든 USART가 병렬로 진행

## 8. 디버깅 — 흔한 실수

| 증상 | 원인 | 해결 |
| --- | --- | --- |
| PING 응답 없음 | 공통 GND 안 됨 | 12V 전원 GND ↔ STM32 GND 연결 |
| PING 응답 없음 | 보드레이트 불일치 | URT-2로 서보 보드레이트 확인 (1Mbps인지) |
| PING 응답 없음 | 외부 풀업 없음 | 1kΩ → +5V |
| 응답 깨짐 | TC 인터럽트 늦음 | UART 속도 너무 빠름, 또는 인터럽트 우선순위 조정 |
| 모터 안 움직임 | Torque Enable=0 | 주소 40에 1 쓰기 |
| 모터가 갑자기 휙 움직임 | Goal Position이 너무 멀어서 | Goal Velocity (46) 또는 Acceleration (41) 제한 |
| 모터 토크 부족 | Torque Limit 낮음 | 주소 48에 800–1000 쓰기 |
| 보행 중 다리 풀림 | Hardware Error 발생 | 주소 65 읽어서 비트 확인, Overload 설정 조정 |
| 12 모터 동기 안 맞음 | 개별 WritePos 사용 | SyncWrite 사용 (0x83) |

## 10. 빠른 참조 카드

```
PING:        FF FF [ID] 02 01 [chk]                 → 6B 응답
READ Pos:    FF FF [ID] 04 02 38 02 [chk]           → 8B 응답
WRITE Byte:  FF FF [ID] 04 03 [Addr] [Data] [chk]   → 6B 응답
WRITE Word:  FF FF [ID] 05 03 [Addr] [DL] [DH] [chk] → 6B 응답

SyncWrite:   FF FF FE [Len] 83 [Addr] [DataPerServo] [ID1][D1...][ID2][D2...] [chk]
             → 응답 없음

Checksum: ~(ID + Len + Inst + Param1 + ... + ParamN) & 0xFF

Position:  0–4095 (12-bit), 1 step ≈ 0.088°, neutral = 2048
Torque:    0–1000 (0.1%), 800 = 80%
Velocity:  -1000 ~ +1000 (step/s)

안전 시작값:
  Max Torque Limit (16) = 800   (80%)
  Torque Limit (48)     = 250   (25%, 캘리브용)
  Overload Torque (36)  = 70    (70%)
  Overload Time (35)    = 50    (500ms)

라디안 변환:
  ticks = 2048 + (rad * 651.9)
  rad   = (ticks - 2048) / 651.9
```
