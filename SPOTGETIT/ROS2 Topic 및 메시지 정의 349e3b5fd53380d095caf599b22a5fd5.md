# ROS2 Topic 및 메시지 정의

## 1. Vender - OAK-D-Lite

## /vendor/camera/image_raw

- **토픽 경로** : `/vendor/camera/image_raw`
- **토픽 정보** : OAK-D Lite RGB RAW 스트림. 480x270, BGR8, 15fps
- **메시지 타입** : `sensor_msgs/Image`
- **발행지** : `vendor/oak_camera_driver`
- **구독지** : `perception/camera_perception`
- **QoS** : queue_size=1, reliability=BEST_EFFORT

## /vendor/camera/encoded

- **토픽 경로** : `/vendor/camera/encoded`
- **토픽 정보** : OAK-D Lite MJPEG 인코딩 스트림. format=jpeg, 15fps
- **메시지 타입** : `sensor_msgs/CompressedImage`
- **발행지** : `vendor/oak_camera_driver`
- **구독지** : `network/image_stream_sender` (통신팀)
- **QoS** : queue_size=1, reliability=BEST_EFFORT

## 3. Perception - Vision

## /perception/camera/victim_detection

- **토픽 경로** : `/perception/camera/victim_detection`
- **토픽 정보** : 인명 탐지 결과. 탐지된 경우에만 발행
- **메시지 타입** : `robot_interfaces/VictimDetection`
- **발행지** : `perception/camera_perception`
- **구독지** : decision, network (미정)
- **QoS** : queue_size=1, reliability=RELIABLE

### 메시지 필드

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `header` | `std_msgs/Header` | 타임스탬프, frame_id |
| `bbox_x` | float32 | bbox 좌측 상단 x (정규화, 0~1) |
| `bbox_y` | float32 | bbox 좌측 상단 y (정규화, 0~1) |
| `bbox_w` | float32 | bbox 너비 (정규화, 0~1) |
| `bbox_h` | float32 | bbox 높이 (정규화, 0~1) |
| `distance` | float32 | 추정 거리 (m), 현재 0.0 고정 |
| `confidence` | float32 | 탐지 신뢰도, 배포 시 제거 예정 |

> bbox 정규화 기준 해상도 : `oak_camera_driver.param.yaml`의 `rgb_width`, `rgb_height`
> 

## 4. Localization - Jetson

## /localization/robot/state

- **토픽 경로** : `/localization/robot/state`
- **토픽 정보** : 로봇 위치 및 방향. 공통 좌표계 기준. 탐지된 경우에만 발행
- **메시지 타입** : `robot_interfaces/RobotLocalization`
- **발행지** : `localization/localization_fusion` (라이다 파트)
- **구독지** : `d_twin` (SIM 파트 - Isaac Sim 위치 반영)
- **QoS** : queue_size=1, reliability=RELIABLE

### 메시지 필드

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `header` | `std_msgs/Header` | 타임스탬프, frame_id |
| `robot_id` | uint8 | 로봇 식별자 (1: robot_01, 2: sim_02, 3: sim_03) |
| `x` | float32 | 공통 좌표계 기준 x (m) |
| `y` | float32 | 공통 좌표계 기준 y (m) |
| `yaw` | float32 | 공통 좌표계 기준 yaw (rad) |

## 5. Planning - PC - 수정필요

## /planning/global_path/robot_{robot#}

- **토픽 경로** : `/planning/global_path/robot_{robot#} (예: robot_01, sim_02, sim_03)`
- **토픽 정보** : 로봇의 global_path. 공통 좌표계 기준.
- **메시지 타입** : `nav_msgs/Path`
- **발행지** : `/planning/global_path_manager` (PC)
- **구독지** : `simulation/sim-executor` (SIM 파트 - 로봇 경로 반영)
- QoS : depth=1, reliability=RELIABLE, durability=TRANSIENT_LOCAL

### 메시지 필드

| **필드** | **타입** | **설명** |
| --- | --- | --- |
| header | std_msgs/Header | 타임스탬프, frame_id (기본값: "map") |
| poses | geometry_msgs/PoseStamped[] | 경로를 구성하는 웨이포인트(위치 x, y, z 및 방향 quaternion) 배열 |
