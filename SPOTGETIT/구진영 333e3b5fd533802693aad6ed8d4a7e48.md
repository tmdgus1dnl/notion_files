## 1. 프로젝트 요약 (Executive Summary)

ROS2 기반의 모바일 로봇 개발자들이 **하드웨어 제어의 복잡함 없이 상위 알고리즘 개발에만 집중**할 수 있도록 돕는 오픈소스 실시간 하부 제어(Low-Level Control) 패키지를 개발한다.

MCU(STM32/ESP32) 환경에서 **FreeRTOS와 C++ HAL을 구축**하고, **micro-ROS를 통해 표준 ROS2 메시지(Twist, Odometry)를 지연 없이 송수신**하는 범용 브릿지 및 디바이스 드라이버 아키텍처를 GitHub에 오픈소스로 배포한다.

> **핵심 철학**: "어떤 ROS2 기반의 상위 제어기(PC, 라즈베리파이 등)를 연결하든, `cmd_vel` 표준 주행 명령만 내리면 실시간으로 바퀴가 굴러가게 만들어주는 범용 하부 제어 패키지"
>

## 2. 개발 배경 및 필요성 (Background & Necessity)

### 문제 인식

| 문제 | 설명 |
| --- | --- |
| **어플리케이션 / 하드웨어 분리 미흡** | 로봇 개발자는 자율주행 알고리즘(AI, SLAM)에 집중해야 하지만, 현실에서는 모터 제어·엔코더 수집·I2C/PWM 통신 등 하드웨어 연동 문제에 많은 시간을 낭비 |
| **표준화된 실시간 제어 인프라 부재** | 시중의 RC카 예제들은 대부분 Python이나 Arduino 기반으로 **실시간성(Real-Time)이 보장되지 않아** 산업용·연구용으로 확장 불가 |
| **하드웨어 종속성** | 특정 보드에 종속된 코드가 많아 보드를 바꾸면 전체를 재작성해야 하는 비효율 발생 |

### 해결 방향

상위 제어기가 무엇이든(PC, SBC 등) 범용적으로 연동되며, **ROS2의 표준 제어 규격을 준수하는 Plug & Play 방식의 실시간 하부 제어 펌웨어 및 브릿지 패키지** 제공

## 3. 핵심 기능 (Key Features)

### 3.1 모듈형 C++ 디바이스 드라이버 및 HAL 구축 (MCU)

- 하드웨어 종속성을 제거한 **범용 HAL 클래스 설계** (Motor, Encoder, IMU 등)
- 특정 보드(STM32 등)용 드라이버를 `/driver` 폴더에 **모듈화**하여 확장성 확보
- 다른 보드를 지원하려면 `/driver/{board_name}` 폴더에 드라이버 하나만 추가하면 전체 시스템 동작

```
/driver
  ├── stm32/     ← 공식 지원
  ├── esp32/     ← 공식 지원
  └── arduino/   ← 커뮤니티 확장 가능
```

### 3.2 FreeRTOS 기반 실시간 스케줄링 (MCU)

- **모터 제어 Task (최고 우선순위)**, 센서 수집 Task, 통신 Task를 분리
- Mutex / Queue를 활용한 Task 간 동기화 설계
- 통신 병목 현상이 하드웨어 구동에 영향을 미치지 않도록 격리

### 3.3 micro-ROS 범용 통신 브릿지 (Middleware)

- ROS2 표준 주행 메시지 **`geometry_msgs/Twist` 구독(Subscribe)** → 모터 PWM으로 변환
- 엔코더 값 읽기 → ROS2 표준 상태 메시지 **`nav_msgs/Odometry` 발행(Publish)**
- Serial / UDP 통신 파이프라인 지원

### 3.4 시스템 성능 검증(Profiling) 및 진단 도구 (Host PC)

- Host-MCU 간 **응답 지연시간(Latency)** 및 **패킷 손실률** 측정 ROS2 진단 노드 제공
- 키보드/조이스틱으로 모터를 테스트하는 **Teleop(수동 제어) 노드** 제공
- 벤치마킹 결과를 README에 수치로 공개 (Python 대비 성능 비교 그래프 포함)

## 5. 사용 기술 스택 (Tech Stack)

| 분류 | 기술 |
| --- | --- |
| **Host 환경** | ROS2 (Humble / Iron), Ubuntu 22.04 |
| **MCU 환경** | STM32 또는 ESP32 (FreeRTOS, micro-ROS 지원 보드) |
| **핵심 언어** | C++14/17 (펌웨어 · HAL · 드라이버) |

| **RTOS** | FreeRTOS |
| **미들웨어** | micro-ROS, DDS (ROS2 기본 통신 레이어) |
| **통신 방식** | Serial (UART), UDP |
| **빌드 도구** | CMake, colcon |
| **개발 도구** | STM32CubeIDE / ESP-IDF, VSCode + PlatformIO |
| **협업 도구** | Git, GitHub (Release · Issues · Wiki) |
| **문서화** | Doxygen, Markdown (영문 README) |

Setup   Core Dev  Core Dev  Sub-Sys  Full-Sys  Profiling Release

### Week 1 — Architecture Setup

- HAL 클래스 **API 설계** (함수명, 데이터 타입 통일)
- GitHub 저장소 생성, 디렉토리 구조 확정 (Host용 / MCU용 분리)
- 브랜치 전략 및 코드 리뷰 프로세스 합의

### Week 2 — Core Development (Low-Level & OS)

| 팀 | 목표 |
| --- | --- |
| **A팀** | 타이머, PWM 설정 및 모터 구동 드라이버 1차 작성 |
| **B팀** | FreeRTOS 구동 테스트 및 micro-ROS Ping 예제 성공 |
| **C팀** | ROS2 Host 환경 구축 및 표준 메시지(`Twist`) 구조 분석 |

### Week 3 — Core Development (Middleware)

| 팀 | 목표 |
| --- | --- |
| **A팀** | 엔코더 센서 데이터 읽기 및 HAL 클래스화 완료 |
| **B팀** | Host에서 보낸 Dummy 데이터를 MCU에서 수신 성공 |
| **C팀** | 키보드 입력을 `Twist` 메시지로 발행하는 테스트 노드 작성 |

### Week 4 — Sub-System Integration

- **[제어-통신 통합]** Host(C팀)에서 내린 Twist 명령 → 미들웨어(B팀) → HAL 함수(A팀) 호출 → **실제 바퀴 구동 확인**

### Week 5 — Full-System Integration (Feedback Loop)

- **[상태 보고 통합]** MCU 엔코더 값(A팀) → 미들웨어(B팀) → Host `Odometry` 토픽(C팀) 수신 확인
- 전체 파이프라인 데이터 흐름 검증 및 디버깅

### Week 6 — Profiling & Tuning

| 팀 | 목표 |
| --- | --- |
| **A/B팀** | 통신 주기 및 Task 우선순위 최적화, PID 제어기 튜닝 |
| **C팀** | 벤치마킹 툴로 시스템 Latency 측정 및 결과 시각화 |
- Python 제어 방식 대비 Latency 비교 수치 확보

### Week 7 — Open Source Release

- 코드 리팩토링 및 **Doxygen 주석** 정리
- **영문 README 작성**: 아키텍처 설명, 설치 방법, API 명세, 성능 지표
- GitHub **Release v1.0.0** 배포 (태그, 릴리즈 노트 포함)
