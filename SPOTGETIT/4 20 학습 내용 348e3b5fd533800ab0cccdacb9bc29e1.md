    # STM32 PWM 모터 제어
    
    ## 1. 모터 종류별 비교
    
    |  | DC모터 | 서보 | BLDC |
    | --- | --- | --- | --- |
    | 제어 | PWM duty cycle | PWM pulse width | ESC |
    | 피드백 | 없음 | 내부 (엔코더/가변저항) | 홀센서 |
    | 복잡도 | 낮음 | 중간 | 높음 |
    | 용도 | 단순구동 | 관절 | 고효율구동 |
    
    ### 서보 모터 내부 구조
    
    ```
    외부 PWM 신호
          ↓
      내부 MCU
          ↓
      목표 각도 계산
          ↓
      엔코더로 현재 각도 읽기
          ↓
      목표 - 현재 = 오차
          ↓
      오차만큼 모터 돌림
          ↓
      오차 = 0 되면 정지  → 피드백 제어
    ```
    
    - 일반 RC 서보 (SG90): 내부 **가변저항** 피드백, 외부 노출 없음
    - 스마트 서보 (STS3215): **자기 엔코더** 피드백, 각도/속도/토크 외부 노출
    
    ### TTL이란?
    
    모터 종류가 아닌 **통신 전압 레벨**이다.
    
    - TTL (5V 기준): HIGH = 5V, LOW = 0V
    - STS3215가 "TTL 통신"이라고 하는 것 = 단거리 저전압 시리얼 통신

    ## 3. PWM 기초
    
    ### 타이머 레지스터
    
    | 레지스터 | 풀네임 | 역할 |
    | --- | --- | --- |
    | PSC | Prescaler | 클럭을 나눠서 느리게 만듦 |
    | ARR | Auto Reload Register | 카운터 주기 결정 (0 ~ ARR → 리셋) |
    | CCR | Capture Compare Register | HIGH 구간 길이 결정 (pulse width) |
    
    ### 50Hz 서보 타이머 설정
    
    ```
    TIM clock: 84MHz
    PSC = 83  → 84,000,000 / (83+1) = 1MHz (1틱 = 1μs)
    ARR = 19999 → 1,000,000 / (19999+1) = 50Hz (주기 20ms)
    ```
    
    > PSC, ARR에 +1 하는 이유: 0부터 시작하므로 실제 개수는 값+1
    > 
    
    ### PWM 파형
    
    ```
    |←────────────── 20ms ──────────────────────→|
    |▔▔▔▔▔▔|__________________________________|
      1~2ms           18~19ms
      HIGH (CCR)       LOW (ARR - CCR)
    ```

    ## 4. DCMotor 클래스
    
    ```cpp
    // Motor.hpp
    #pragma once       // 중복 include 방지
    #include "tim.h"   // HAL 타이머 함수 선언
    
    class DCMotor {
    public:
        // 생성자: 이니셜라이저 리스트로 멤버변수 초기화
        DCMotor(TIM_HandleTypeDef* htim, uint32_t channel)
            : htim_(htim), channel_(channel), speed_(0) {}
    
        // HAL_Init() 이후 호출해야 해서 생성자와 분리
        void init() {
            HAL_TIM_PWM_Start(htim_, channel_);
        }
    
        void setSpeed(uint8_t percent) {
            if (percent > 100) percent = 100;       // clamp (불변식 보호)
            speed_ = percent;                        // 현재 상태 저장 (대입)
    
            uint32_t arr = __HAL_TIM_GET_AUTORELOAD(htim_);  // ARR 레지스터 읽기
            // arr+1: 카운터가 0~ARR이라서 실제 틱 수는 ARR+1
            uint32_t pulse = (arr + 1) * percent / 100;      // duty cycle → 틱 변환
            __HAL_TIM_SET_COMPARE(htim_, channel_, pulse);   // CCR에 write
        }
    
        void stop() { setSpeed(0); }  // 가독성용 래퍼
    
        uint8_t getSpeed() const { return speed_; }  // const: 멤버변수 변경 안 함을 보장
    
    private:
        TIM_HandleTypeDef* htim_;  // 포인터: 원본 HAL 구조체를 가리켜야 함
        uint32_t channel_;
        uint8_t speed_;
    };
    ```
    
    ### setSpeed() 동작 원리
    
    ```
    percent = 75, ARR = 999 일 때:
    
    pulse = (999 + 1) * 75 / 100 = 750
    
    PWM 파형:
    |▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔|_______________|
    0              750            999
          HIGH          LOW
          75%           25%
    ```

    ## 5. Servo 클래스
    
    ```cpp
    // Servo.hpp
    #pragma once
    #include "tim.h"
    
    class Servo {
    public:
        Servo(TIM_HandleTypeDef* htim, uint32_t channel)
            : htim_(htim), channel_(channel), angle_(90) {}
            // angle_(90): 중립 위치로 초기화
    
        void init() {
            HAL_TIM_PWM_Start(htim_, channel_);
            setAngle(90);  // 하드웨어가 어느 위치에 있을지 모르니까 명시적으로 중립 이동
        }
    
        void setAngle(uint8_t deg) {
            if (deg > 180) deg = 180;    // clamp
            angle_ = deg;                 // 현재 상태 저장
    
            // (uint32_t) 캐스팅: deg(uint8_t) * 1000 = 최대 180000 → uint8_t 오버플로우 방지
            uint32_t pulse = 1000 + (uint32_t)deg * 1000 / 180;
            __HAL_TIM_SET_COMPARE(htim_, channel_, pulse);
        }
    
        uint8_t getAngle() const { return angle_; }
    
    private:
        TIM_HandleTypeDef* htim_;
        uint32_t channel_;
        uint8_t angle_;
    };
    ```
    
    ### setAngle() 동작 원리
    
    서보 표준: 1ms = 0도, 2ms = 180도 (제조사 약속)
    
    ```
    1틱 = 1μs이므로:
      0도   → 1000틱 = 1ms
      90도  → 1500틱 = 1.5ms
      180도 → 2000틱 = 2ms
    ```
    
    공식 분해:
    
    ```
    pulse = 1000 + deg * 1000 / 180
    
    1000          → 0도일 때 기본 offset (1ms)
    deg * 1000 / 180 → 각도를 0~1000틱 범위로 비례 변환
    ```

    ## 6. DCMotor vs Servo 비교
    
    |  | DCMotor | Servo |
    | --- | --- | --- |
    | 제어 대상 | 속도 (0~100%) | 각도 (0~180도) |
    | CCR 단위 | duty cycle (비율) | pulse width (절대 시간) |
    | 초기값 | speed_(0) | angle_(90) |
    | init() | PWM 시작만 | PWM 시작 + 중립 이동 |
    | 핵심 공식 | `(arr+1) * percent / 100` | `1000 + deg * 1000 / 180` |
    
    **duty cycle vs pulse width**
    
    - duty cycle: 비율(%), "전체 주기 중 HIGH가 몇 %"
    - pulse width: 시간(ms), "HIGH가 몇 ms 동안 지속"
    - 서보가 pulse width를 쓰는 이유: 5~10% 범위로 너무 좁아서 정밀도 부족, 타이머 설정과 무관한 절대값 필요
