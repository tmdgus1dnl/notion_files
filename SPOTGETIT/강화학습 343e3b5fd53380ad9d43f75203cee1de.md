    effort를 STS32115 30kg 에 맞게 설정(2.94N), 추후 하드웨어 설계 완료 후 joint 위치, 정밀 무게 변경 정도만 하면 되도록
    - default 관절 각도 설정, joint의 swiffness,damping(PD제어의 Kp,Kd 해당값) 설정, action scale(관절이 움직이는 최대 각도) 설정
    - 종료(실패) 조건 설정 - base link 높이, 기울어진 정도
    - 설정 URDF 기준으로 걸을 때 토크 포화 발생
        - 하드웨어적 한계(낮은 effort, Kp, Kd를 높게 줄 수 없는 상황)
        - `torque = Kp × (목표위치 - 현재위치) - Kd × 속도`
        - 낮은 동작 토크를 위해 Kp,Kd를 낮추면 관절이 버티지 못하고 무너짐
        - 초기 action scale = 2.5, Kp를 한계 이상으로 낮출 수 없으니 action scale을 줄여 공식의 (목표위치 - 현재위치) 값을 줄이자 → scale = 1.2로 줄임
        - 초기 설정한 2.5 값은 기존 legged_gym 에서 Anymal 기준, AnyMal은 spot micro에 비해 사이즈도 크고(다리 하나 길이가 50cm) 더 무거운 무게 기준
        - 최종 → swiffness=20, damping=0.5, action scale=1.2로 설정. 이래도 토크 포화율 50.3%(기존 거의 80%에 육박했음)
    - legged_gym 에 세팅되어 있는 설정이 spot micro 같은 소형이 아닌 더 큰 모델 기준인게 있었음. 이런 거 하나하나 확인해서 spot micro 사이즈, 성능(토크) 기준에 맞춰야 함\
    - 현재 설정 기준 토크 포화율 50.3%(토크가 최대한계에 도달한 정도인듯?), 특히 어깨, 상완 쪽 모터의 포화비율이 약 70~75% 정도로 높다
    - 관절한계는 안정권
    - 평균 x속도 오차 : 0.1163m/s → 줄일 필요가 있음(iteration 늘리거나 reward를 조정)
        - 아직 초기 다른 reward를 추가하지 않고 하나씩 잡아가는 과정이라 iteration도 500으로 작게 준 상태. 고려했을 때 괜찮을수도?
    - 평균 y속도 오차 : 0.0618m/s → 현재 학습 때 control로 x속도만 줬음. 이 단계에서 줄일 필요가 있는지 검증 필요
    - 조기 종료(넘어짐) 비율 1.5%, 자세 안정성은 평균높이 0.22m, 표준편차 0.0028로 안정적
    - legged_gym 에서 직선속도 0.2m/s 이하는 0으로 처리한다고 함(이 때문에 대부분의 속도 범위에서 속도가 0으로 설정되어 걷기 학습이 아니라 서있는 걸로 학습됨) → 0.25~0.4 로 학습 범위 수정
        - spot micro에서 한 걸음당 속도가 최소 0.2m/s 인지 확인 필요.
- 2주차 작업내용
    - URDF 설정이 많이 중요하다.
        - URDF상 정면이 -x방향이었다. legged_gym은 +x 방향이 정면이라 직진명령 내리면 뒤로는 문제 발생
        - axis의 의미 - 서보모터 축이 보고있는 방향이 아니라 action의 방향. 즉 다리를 핀다 접는다 같은 행위가 +, - 로 정해지는 거라 서보가 대칭으로 좌우를 다르게 보고 있더라도 같은 축으로 놔야함
        - 어깨축의 서보모터는 다 +x 방향을 봐서 axis를 1 0 0 으로 뒀었는데 그러면 똑같이 +0.1을 줬을 때 왼쪽어깨는 바깥으로, 오른쪽어깨는 안쪽으로 꺾임 → 오른쪽 어깨 axis를 -1 0 0 으로 둬야함
