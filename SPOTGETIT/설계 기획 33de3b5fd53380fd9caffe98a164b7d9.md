> 실물 1대와 가상 N대의 혼합 군집 구조를 통해, 다중 구조정찰 로봇의 임무 분담·탐색·상태 관제를 디지털트윈 기반으로 검증하는 시스템

## 3. 디지털 트윈

### 목적

- 실제 험지 환경 구현이 금전적·환경적으로 불가능하므로, 시뮬레이션에서 산지 등 극한 환경을 학습
- 실제 시연은 평지 + 장애물 수준으로 진행 (Sim-to-Real 갭 존재)

### 플랫폼

- **시뮬레이터** : Gazebo
- **미들웨어** : ROS2 (Ubuntu 22.04 LTS 기준으로 실물·시뮬 환경 통일)

### 역할

| 목적 | 내용 |
| --- | --- |
| Sim-to-Real | 시뮬에서 험지 학습 → 실제 환경에 배포 |
| RL 학습 | 단일 로봇 주행 RL + 다중 로봇 경로 조율 MARL |
| 실시간 모니터링 | 로봇 ↔ 관제 시스템 연동 |

---

## 4. 로봇 역할 구조

- 전 로봇 **동일 임무**
- 각자 **다른 경로**로 출발하여 병렬 탐색 수행
- 로봇 간 중앙 조율 없이 독립 운용 (경로 중복 방지는 MARL로 처리)

| --- | --- |

## 6. Jetson Orin Nano Task 분배

| Task | 담당 | 비고 |
| --- | --- | --- |
| LiDAR Odometry (KISS-ICP) | Orin Nano | 맵 생성 없이 궤적만 추정 |
| Navigation (경로 계획·장애물 회피) | Orin Nano |  |
| Vision AI (인명 탐지) | Orin Nano | TensorRT 경량화 필수 |
| 보행 제어 (관절 저수준) | MCU | Orin Nano 연산 부담 분리 |

> **Vision AI 추론 주기** : 험지 특성상 이동 속도가 느리므로 낮은 FPS로도 충분. 연산 여유 확보 가능.

## 7. 위치 추정 및 관제 표시

### 방식

- **LiDAR Odometry** (SLAM 아님) 사용
- 맵 생성·저장 오버헤드 없음
- 출발점 기준 상대 좌표(pose)를 누적하여 궤적 생성

### 관제 표시

- 로봇이 주기적으로 pose를 관제로 송신
- 관제는 수신한 좌표를 순서대로 **실선으로 연결하여 경로 표시**
- 다중 로봇은 로봇별 색상으로 구분
- 현재 위치 = 실선의 끝점 (출발점 기준 상대 좌표)

> **한계** : 루프 클로저 없음. 장시간 운용 시 오차 누적. 정밀 위치 파악이 아닌 **대략적 이동 경로 확인** 용도로 사용.
> 

### 관제 수신 데이터 목록

- 위치 (LiDAR Odometry pose)
- 카메라 피드
- 인명 탐지 결과
- 배터리 상태

## 8. RL 구조

| 레이어 | 목적 | 프레임워크 |
| --- | --- | --- |
| 단일 로봇 주행 RL | 보행 + 장애물 회피 | Stable-Baselines3 |
| 다중 로봇 경로 조율 MARL | 경로 중복 방지, 탐색 효율 최대화 | RLlib |

> **진행 순서** : 단일 로봇 주행 RL 완성 후 MARL 단계로 진입.
> 

### Sim-to-Real 전략

- 시뮬에서 다양한 환경으로 학습 → 실제 환경 적용
- **도메인 랜덤화 (Domain Randomization)** 적용 필요
