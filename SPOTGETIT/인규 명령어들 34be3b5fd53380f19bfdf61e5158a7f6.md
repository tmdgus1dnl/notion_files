## 테스트 원리

mock 스크립트는 safety_supervisor가 필요로 하는 5개 토픽을 publish합니다.

```
/control/cmd_vel_raw/spot_01       v=0.2, w=0.1 고정
/navigation/state/spot_01          시나리오별 nav_state
/perception/lidar/obstacle_model   시나리오별 전방 거리
/perception/lidar/free_space_model path_available=true 고정
/localization/pose                 (1.0, 1.0) 고정
```

safety_supervisor는 이 데이터를 받아 분기별 처리 후 `/control/cmd_vel`로 최종값을 내보냅니다.

## 시나리오별 예상 결과

| scenario | 조작 내용 | 예상 결과 |
| --- | --- | --- |
| `normal` | 장애물 없음, 모든 데이터 정상 | `linear.x ≈ 0.2`, `angular.z ≈ 0.1` (가속 제한으로 점진적 수렴) |
| `emergency` | front.nearest = 0.20m (emergency_stop=0.30 이내) | `linear.x = 0`, `angular.z = 0` 즉시 |
| `soft_stop` | front.nearest = 0.50m (soft_stop=0.65 이내) | `linear.x < 0.2` 감속, `angular.z ≈ 0.1` 통과 |
| `cmd_stale` | cmd_vel_raw publish 중단 | `linear.x`, `angular.z` 점진 감속 후 0 |
| `pose_stale` | pose publish 중단 | `linear.x`, `angular.z` 점진 감속 후 0 |
| `perception_stale` | obstacle publish 중단 | `linear.x`, `angular.z` 점진 감속 후 0 |
| `goal_reached` | nav_state = NAV_GOAL_REACHED | `linear.x`, `angular.z` 점진 감속 후 0 |

---

## 분기별 처리 원리

```
우선순위 순으로 판단

1. cmd_stale / pose_stale / perception_stale / goal_reached
   → target v=0, w=0 설정
   → acceleration limit으로 점진적으로 0에 수렴
   → 데이터 복구 시 정상 재개

2. emergency_stop (front_clearance < 0.30m)
   → v=0, w=0 즉시 (acceleration limit 무시)

3. soft_stop (front_clearance < 0.65m)
   → v = input_v * (front_clearance / soft_stop_distance_m)
   → w는 그대로 통과

4. max speed 제한
   → v = clamp(v, 0, max_safe_linear_x_mps)
   → w = clamp(w, -max_safe_angular_z_radps, max_safe_angular_z_radps)

5. acceleration limit
   → 가속 : v_out = prev_v + max_linear_accel * dt
   → 감속 : v_out = prev_v - max_linear_decel * dt
   → w도 동일하게 max_angular_accel 적용
```

### normal 시나리오 상세

cmd_vel_raw가 v=0.2로 들어오더라도 처음에는 prev_v=0 이므로 acceleration limit에 의해 점진적으로 수렴합니다.

```
tick 1 : prev_v=0.00 → out_v = 0.00 + 0.33*0.1 = 0.033
tick 2 : prev_v=0.033 → out_v = 0.033 + 0.033 = 0.066
...
약 6tick(0.6s) 후 : out_v ≈ 0.2 수렴
```

### [2] approach_and_clear

**설명** : 장애물 감지 후 사라지는 흐름 (clear hold time 포함)

**주입 데이터 흐름**

| Step | nearest (m) | 유지 | 예상 FSM | 비고 |
| --- | --- | --- | --- | --- |
| 1 | 9999 | 2s | TRACKING (1) |  |
| 2 | 0.69 | 2s | PLANNING (2) | blocked 진입 |
| 3 | 0.50 | 2s | PLANNING (2) |  |
| 4 | 0.86 | 1s | PLANNING (2) | clear 조건 충족, hold 시작 |
| 5 | 0.86 | 1s | TRACKING (1) | hold 0.5s 경과 후 복귀 |

# 로컬PC ROS2

## **global_path_manager**

맵(map.yaml)을 읽어 path set을 생성하고, 겹침 최소 + 커버리지 최대 기준으로 num_robots개 경로를 선택해 발행.실행 후 종료. 맵 변경 시 재실행.

### **빌드**

```jsx
cd ~/robot_ws
colcon build --packages-select global_path_manager
```

### **실행**

```jsx
source install/setup.bash
ros2 launch global_path_manager global_path_manager.launch.py
```

### **맵 변경 시**

```jsx
vim ~/robot_ws/src/planning/global_path_manager/config/map.yaml

cd ~/robot_ws
colcon build --packages-select global_path_manager
source install/setup.bash
ros2 launch global_path_manager global_path_manager.launch.py
```

### **경로 확인**

```jsx
source install/setup.bash
ros2 topic echo --once /planning/global_path/robot_01
ros2 topic echo --once /planning/global_path/sim_02
ros2 topic echo --once /planning/global_path/sim_03
```

## **sim_executor**

global_path를 수신해 sim_02, sim_03을 0.1m/s로 이동시키며 /localization/robot/state 발행.global_path_manager가 먼저 실행되어 있어야 함.

### **빌드**

```jsx
cd ~/robot_ws
colcon build --packages-select sim_executor
```

### **실행**

```jsx
source install/setup.bash
ros2 launch sim_executor sim_executor.launch.py
```

### **위치 확인**

```jsx
ros2 topic echo /localization/robot/state
```
