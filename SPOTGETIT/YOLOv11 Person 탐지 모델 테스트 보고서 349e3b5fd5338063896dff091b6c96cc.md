## 4. TensorRT Engine 변환 및 Orin Nano 추론 테스트

### 4-1. INT8 변환 시도 → 실패

```bash
trtexec --onnx=best.onnx --saveEngine=best.engine --int8 --memPoolSize=workspace:512M
```

**결과** : 엔진 변환은 성공했으나 추론 시 conf ≈ 0으로 탐지 전혀 되지 않음

**원인** : INT8 양자화는 캘리브레이션 데이터 없이 변환하면 정밀도가 크게 손실됨

### 4-2. FP16 변환 → 성공

```bash
trtexec --onnx=best.onnx --saveEngine=best_fp16.engine --fp16 --memPoolSize=workspace:512M
```

## 6. 한계 및 향후 계획

### 한계

- FP16 기준 conf가 프레임마다 크게 변동 (0.0 ~ 0.79)
- 사람이 있어도 탐지 못하는 프레임 존재 → 모델 일반화 부족
- INT8은 캘리브레이션 없이 사용 불가

### 향후 계획

- 추가 수집 데이터셋 (v3, 약 5100장) 적용 후 재학습
- OAK-D Lite 직접 촬영 데이터로 파인튜닝
- INT8 캘리브레이션 데이터 구성 후 재시도 검토
