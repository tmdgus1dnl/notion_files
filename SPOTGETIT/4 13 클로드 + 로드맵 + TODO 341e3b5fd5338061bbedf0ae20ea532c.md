## 대화 1: STM32 핀 단락 및 STS3215 CAD 파일

### STM32 핀이 금속에 닿으면 쇼트 나는가?

- **위험한 경우**: GPIO 출력 모드(Push-Pull)에서 HIGH 출력 핀이 GND 금속에 닿으면 단락 → 핀/포트 드라이버 손상 가능. 두 출력 핀이 금속으로 연결되어 하나가 HIGH, 하나가 LOW일 때도 위험
- **괜찮은 경우**: 입력 모드(Hi-Z) 핀이 GND에 닿으면 그냥 LOW로 읽힐 뿐. 플로팅 금속(어디에도 연결 안 된)에 닿으면 문제 없음
- **실무 팁**: Nucleo-64 보드 뒷면 납땜 포인트 노출 주의. 비전도성 매트/스페이서 사용. 3S LiPo 전원 OFF 상태에서 배선하고 통전 전에 쇼트 체크

## 대화 2: Git Flow 전략으로 레포 구성

### 브랜치 구조

```
main  ── 프로덕션 배포 (직접 푸시 불가, dev → MR 필수)
  └── dev  ── 스테이징 통합 (직접 푸시 불가, feat/* → MR 필수)
        └── feat/S14P21A206-{이슈번호}
```

### 커밋 메시지 컨벤션

```
type(scope): subject #S14P21A206-{이슈번호}
```

| type | 설명 |
| --- | --- |
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| docs | 문서 수정 |
| refactor | 코드 리팩토링 |
| deploy | 배포 관련 설정 수정 |
| chore | 빌드, 패키지, 설정 |
| remove | 불필요한 파일/코드 삭제 |
| test | 테스트 코드 추가/수정 |
- scope: `fe`, `be`, `ai`, `mock`, `data`, `infra`, 여러 개면 `all`

### 작업 흐름 요약

1. `dev`에서 `feat/S14P21A206-{이슈번호}` 브랜치 생성
2. 작업 후 커밋 (매 커밋에 이슈번호 포함)
3. 푸시 → GitLab MR 생성 (Source: feat → Target: dev)
4. 리뷰 반영 후 머지 (Delete source branch 체크)
5. 로컬 정리: `dev` pull + feat 브랜치 삭제

### MR 본문 템플릿

```markdown
## 작업 내용
- 구현 내용 기술

## Jira
close #S14P21A206-{이슈번호}

## 스크린샷

## 로컬 환경 테스트
| Given(조건) | When(행동) | Then(결과) |
|---|---|---|
```

### 자주 쓰는 명령어

- **dev 충돌 해결**: feat 브랜치에서 `git pull origin dev` → 충돌 해결 → 커밋/푸시
- **작업 전환**: `git stash` → 다른 브랜치 작업 → `git stash pop`
- **커밋 합치기**: `git rebase -i HEAD~{수}` → pick → squash

## 대화 3: SpotMicro 하드웨어 제작 (현재 대화)

### 3-1. CAD 설계 환경 및 파일 포맷

### 파일 포맷 비교

| 포맷 | 특성 | Fusion 360 |
| --- | --- | --- |
| STL | 메쉬(삼각형), 히스토리 없음, 면 개념 없음 | Insert Mesh |
| STEP | B-Rep, 서피스 정보 유지, 히스토리 없음 | File → Open 또는 Data Panel |
| FCStd | FreeCAD 네이티브, 히스토리 완전 보존 | 직접 못 열음 → STEP 익스포트 필요 |
| f3d | Fusion 360 네이티브, 히스토리 보존 | 바로 열기 가능 |

### STL 임포트 시 주의사항

- 단위 문제: STL이 mm 단위인데 Fusion이 cm으로 읽으면 10배 커짐
- 실제 사례: 브래킷이 1499mm(1.5m)로 임포트됨 → 정상은 약 150mm
- 해결: Insert Mesh 재임포트 시 Unit Type을 Millimeter로 지정, 또는 Scale Mesh에서 0.1 적용
- STL 메쉬 구멍 채우기: Mesh 탭 → Modify → Fill Hole (버전에 따라 없을 수 있음)

### 여러 파일 조합

- Insert → Insert Mesh 반복으로 STL 여러 개 추가
- STEP/f3d는 Data Panel Upload 후 Insert into Current Design, 또는 드래그 앤 드롭
- 각 파트를 별도 Component로 관리
- 이동/회전: 단축키 M, Move Object 드롭다운에서 Bodies 선택

### Fusion 360 기본 조작

- 화면: 가운데 휠 드래그 = 패닝, Shift + 가운데 = 오빗, 휠 스크롤 = 줌
- 모델링: 스케치(2D) → 피처(3D) 순서
- 단축키: E(Extrude), M(Move), L(Line), R(Rect), C(Circle), D(Dimension), S(검색)
- 스케치 색상: 검은색 = 완전 구속, 파란색 = 자유도 남음

### 3-2. STS3215 서보 모터

- 외형: 약 40 × 20 × 40.5mm (Bounding Box 45.40 × 24.80 × 36.80mm)
- 스톨 토크: **30 kg·cm @ 12V**
- 통신: Half-duplex UART, 1Mbps
- 무게: 약 65g
- SpotMicro 원본(SG90/MG996R 기준)과 맞지 않아 마운트 브래킷 신규 설계 필요

### 3-3. 토크 분석 및 다리 설계

### 총 중량: 약 1.5~2kg

| 부품 | 무게 |

| STS3215 × 12 | 780g |
| PLA 프레임 | 300~500g |
| Jetson Orin Nano | 200g |
| STM32 Nucleo-64 | 30g |
| 3S LiPo | 200~400g |
| 배선/패스너 | 100g |

### 다리 구성: arm 15cm + wrist 15cm = 전체 30cm

### 토크 분석 (12V, 스톨 30 kg·cm)

| 보행 상태 | 다리당 하중 | 필요 토크 | 스톨 대비 |
| --- | --- | --- | --- |
| 4다리 정적 | 500g | 15 kg·cm | 50% |

| 트롯 게이트 (2다리) | 1000g | 30 kg·cm | **100%** |
- 착지 충격: 정적의 1.5~2배 → 트롯 시 스톨 초과 가능
- 연속 운용 토크: 스톨의 50~70% (15~21 kg·cm)

### 3-4. 보행 제어: 고전제어 + RL 하이브리드

### 아키텍처

- Jetson: RL 정책 추론, IMU/비전 입력 → 목표 관절 각도 출력
- STM32: PD 컨트롤러로 목표 각도 추종, 토크/관절 리밋 클램핑

### 개발 순서

1. IK 기반 고전제어 보행 (크리핑 게이트)
2. 시뮬레이션 환경 (MuJoCo/Isaac Gym, URDF)
3. RL 정책 학습 (액션 = 관절 각도 오프셋, 하위 PD 유지)

### 3-5. 전력 시스템

### 구성

```
3S LiPo (11.1V, 2200mAh+, 25C+)
  ├── PDB 메인 패드 → 서보 12V 버스 (star topology)
  ├── Buck-Boost (5V 5A) → Jetson Orin Nano
  └── 3.3V 레귤레이터 or Jetson USB → STM32
```

- FPV 드론 PDB (XT60, 약 7,800원) 사용 가능
- PDB 내장 5V BEC는 2~3A로 Jetson(피크 3A)에 부족 → 별도 buck-boost 필수
- Jetson 직접 12V 연결: 스펙상 가능(9~20V)이나 voltage sag 위험 → buck-boost 권장

### 3-6. 서보 통신: 74HC126 버퍼 IC

### 필요성

- STS3215 half-duplex UART에서 서보 응답 패킷(위치 피드백) 수신 시 버스 충돌 방지
- STM32 native half-duplex보다 하드웨어 방향 제어가 안정적

### 배선 구조

```
STM32 TX → 74HC126 CH1 (1A→1Y) → 서보 DATA
STM32 RX ← 74HC126 CH2 (2Y←2A) ← 서보 DATA

GPIO (DIR) → CH1 OE (TX enable)
           → CH2 OE (RX enable, 반전 필요)
```

- GPIO HIGH = 송신, GPIO LOW = 수신
- 74HC241 사용 시 active-low OE 내장으로 인버터 불필요
- VCC = 3.3V, DATA 라인에 10kΩ 풀업
- 4채널 UART → 74HC126 **2개** 필요
