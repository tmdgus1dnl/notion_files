# 고전/RL 전환 연동

결론부터 말하면, jyh/SPOT_GET_IT 안에는 이미 고전 제어 ROS2 노드와 RL/고전/stand 전환 mux 로직이 있
습니다. 다만 현재 사용 중인 것으로 보이는 /home/jetson/robot_ws에는 그 고전 제어 패키지가 아직 없
고, mux도 RL/STAND까지만 지원합니다.

찾은 구조

- 고전 보행 노드: jyh/SPOT_GET_IT/robot_ws/src/control/classical_gait/classical_gait/
classical_gait_node.py:58
    - /cmd_vel 구독
    - /control/classical/joint_target 50Hz publish
    - open-loop IK 기반 trot, ONNX/RL 없음
    - GaitPhaseGenerator + TrotIkReference 사용
    - 출력 JointTarget.mode = 3
- 전환 mux: jyh/SPOT_GET_IT/robot_ws/src/control/motion_manager/src/joint_target_mux_node.cpp:34
    - /control/behavior/mode 구독
    - 허용 모드: RL, STAND, CLASSICAL
    - 입력:
        - /control/rl/joint_target
        - /control/stand/joint_target
        - /control/classical/joint_target
    - 출력:
        - /control/selected/joint_target
    - RL/classical target이 stale이면 STAND로 fallback
- 설정에도 전환 명령이 적혀 있음: jyh/SPOT_GET_IT/robot_ws/src/control/motion_manager/config/
motion_manager.param.yaml:27

ros2 topic pub /control/behavior/mode std_msgs/msg/String "data: 'CLASSICAL'" --once
ros2 topic pub /control/behavior/mode std_msgs/msg/String "data: 'RL'" --once
ros2 topic pub /control/behavior/mode std_msgs/msg/String "data: 'STAND'" --once

주의할 점

- /home/jetson/robot_ws에는 classical_gait 패키지가 없습니다. /home/jetson/robot_ws/src/control/
motion_manager의 mux도 RL/STAND만 허용합니다: robot_ws/src/control/motion_manager/src/
joint_target_mux_node.cpp:86
- jyh/SPOT_GET_IT의 bringup 런치도 classical_gait_node를 같이 띄우지는 않습니다. 별도 런치 파일만
있습니다: jyh/SPOT_GET_IT/robot_ws/src/control/classical_gait/launch/classical_gait.launch.py:17
- 현재 /home/jetson/robot_ws의 RL은 /control/cmd_vel/spot_01을 구독합니다: robot_ws/src/control/
rl_locomotion/rl_locomotion/rl_locomotion_node.py:292. 반면 jyh/SPOT_GET_IT의 고전 보행은 /
cmd_vel을 구독하므로, 실제 통합 시 토픽 remap 또는 수정이 필요합니다.
- JointTarget.msg에는 MODE_CLASSICAL이 없고 3번은 MODE_CROUCH입니다: jyh/SPOT_GET_IT/robot_ws/src/
interfaces/robot_interfaces/msg/JointTarget.msg:3. 다만 actuator bridge는 mode 3을 OPERATE로 매핑
해서 target 자체는 전달됩니다: jyh/SPOT_GET_IT/robot_ws/src/control/actuator_bridge/src/
actuator_bridge_node.cpp:64

따라서 새로 전환 로직을 처음부터 만들기보다는, jyh/SPOT_GET_IT의 classical_gait 패키지와 확장된
motion_manager mux를 기준으로 /home/jetson/robot_ws에 맞게 가져오는 쪽이 맞아 보입니다. 핵심 작업은
classical_gait 추가, mux를 CLASSICAL 지원 버전으로 교체/반영, cmd_vel 토픽 정합, bringup에
classical 노드 포함 정도입니다.
