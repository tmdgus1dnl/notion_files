# 전체 도구 정리

---

### 디버깅 도구

| 도구 | 잡는 것 | 사용법 |
| --- | --- | --- |
| **TSan** | Race condition | `gcc -fsanitize=thread -g` |
| **ASan** | 메모리 오염, overflow | `gcc -fsanitize=address -g` |
| **helgrind** | 동기화 버그, lock 순서 | `valgrind --tool=helgrind ./app` |
| **memcheck** | 메모리 누수, use-after-free | `valgrind --leak-check=full ./app` |
| **scan-build** | 포인터 위험 코드 정적 분석 | `scan-build gcc -o app main.c` |
| **ipcs** | 좀비 공유 메모리 확인 | `ipcs -m` / `ls /dev/shm` |
| **GDB** | 크래시 위치 추적 | `gdb ./app core` |

```bash
# 개발 중엔 TSan + ASan 같이 켜기
gcc -fsanitize=thread,address -g -o app main.c
```

### 테스트 도구

| 도구 | 용도 | 사용법 |
| --- | --- | --- |
| **Mock Receiver** | Jetson 없이 가짜 데이터 주입 | 직접 구현 |
| **Stress Test** | 강제로 race condition 유발 | 직접 구현 |
| **iperf3** | 무선 구간 처리량 측정 | `iperf3 -c [IP] -u -b 10M` |
| **tcpdump** | 실제 패킷 캡처, 손실 확인 | `tcpdump -i wlan0 -w dump.pcap` |
| **ping** | 기본 지연 측정 | `ping -i 0.1 [IP]` |

```bash
테스트 순서:
  단위 테스트  →  Mock Receiver + TSan
  동기화 테스트 →  Stress Test + helgrind
  네트워크 테스트 →  iperf3 + tcpdump
  통합 테스트  →  실제 Jetson 연결
```

### 성능 측정 도구

| 도구 | 측정 항목 | 사용법 |
| --- | --- | --- |
| **코드 내 타임스탬프** | latency, jitter | `clock_gettime()` |
| **CSV 기록** | 시계열 데이터 저장 | `fprintf(f, "%lu,%lu\n", ...)` |
| **iperf3** | throughput, 손실률 | `--logfile result.txt` |
| **matplotlib** | 그래프 시각화 | `python plot.py metrics.csv` |
| **htop** | CPU/메모리 실시간 | `htop` |
| **nethogs** | 프로세스별 네트워크 사용량 | `nethogs wlan0` |

### 언제 뭘 쓰냐

```bash
개발 중
  └─ TSan + ASan 항상 켜기
  └─ Mock Receiver로 기능 확인

기능 완성 후
  └─ helgrind + memcheck 최종 검증
  └─ Stress Test로 동기화 한계 테스트

네트워크 붙인 후
  └─ iperf3로 무선 구간 처리량
  └─ 코드 내 타임스탬프로 latency 기록
  └─ CSV → matplotlib으로 개선 전후 비교

배포 직전
  └─ ipcs로 좀비 shm 없는지 확인
  └─ tcpdump로 실제 패킷 흐름 확인
```
