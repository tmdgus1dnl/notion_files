    ## 1. 먼저 통신 정상 확인
    
    다른 터미널에서:
    
    ```bash
    cd ~/robot_ws
    source install/setup.bash
    ```
    
    ### actuator 상태 확인
    
    ```bash
    ros2 topic echo /control/actuator/status --once
    ```
    
    정상 기준은 대략:
    
    ```yaml
    status: 0
    fault_code: 0
    spi_connected: true
    servo_connected: true
    ```
    
    `torque_enabled`는 STM의 torque 처리 방식에 따라 STAND/OPERATE 명령이 들어간 뒤 true가 될 수 있습니다.
    
    ### feedback 주기 확인
    
    ```bash
    ros2 topic hz /control/actuator/joint_feedback
    ros2 topic hz /control/actuator/imu
    ```
    
    둘 다 50Hz 근처면 좋습니다.
    
    ### selected target 확인
    
    ```bash
    ros2 topic hz /control/selected/joint_target
    ```
    
    이것도 50Hz 근처여야 합니다.
    
    ---
    
    ## 2. STAND 모드 확인
    
    기본이 STAND라서 사실 따로 안 줘도 됩니다. 그래도 명시적으로 주려면:
    
    ```bash
    ros2 topic pub --once /control/behavior/mode std_msgs/msg/String "{data: 'STAND'}"
    ```
    
    이 상태에서 로봇이 기본 자세를 유지하는지 봅니다.
    
    확인할 것:
    
    ```
    - 관절 순서가 맞는지
    - 좌우 다리 방향이 반대로 움직이지 않는지
    - 떨림이 심하지 않은지
    - 서보 과열/과전류가 없는지
    - /control/actuator/status가 OK인지
    ```
    
    이 단계에서 이상하면 RL로 넘어가면 안 됩니다.
    
    ---
    
    ## 3. RL 모드로 전환
    
    STAND가 안정적이면 RL로 바꿉니다.
    
    ```bash
    ros2 topic pub --once /control/behavior/mode std_msgs/msg/String "{data: 'RL'}"
    ```
    
    그 다음 바로 속도를 주지 말고, 먼저 `cmd_vel=0`을 줍니다.
    
    ```bash
    ros2 topic pub --once control/cmd_vel/spot_01 geometry_msgs/msg/Twist \
    "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
    
    ros2 topic pub --rate 10 control/cmd_vel/spot_01 geometry_msgs/msg/Twist \
    "{linear: {x: 0.07, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
    ```
    
    확인:
    
    ```bash
    ros2 topic echo /control/rl/debug --once
    ros2 topic hz /control/rl/joint_target
    ros2 topic hz /control/selected/joint_target
    ```
    
    RL 모드에서 `selected_joint_target`이 계속 나오면 mux까지는 정상입니다.
    
    ---
    
    ## 4. 공중 RL 저속 테스트
    
    처음은 아주 작게 합니다.
    
    ```bash
    ros2 topic pub --rate 10 /cmd_vel geometry_msgs/msg/Twist \
    "{linear: {x: 0.03, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
    ```
    
    멈추려면 `Ctrl+C`로 publish 중단 후, zero cmd를 한 번 보냅니다.
    
    ```bash
    ros2 topic pub --once /cmd_vel geometry_msgs/msg/Twist \
    "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
    ```
    
    괜찮으면 다음 단계:
    
    ```bash
    ros2 topic pub --rate 10 /cmd_vel geometry_msgs/msg/Twist \
    "{linear: {x: 0.05, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
    ```
    
    초기에는 `0.03 → 0.05 → 0.08 → 0.10` 정도로 천천히 올리세요.

    ## 5. yaw 테스트
    
    처음부터 yaw를 넣지 말고, 직진이 되는 걸 본 뒤 합니다.
    
    ```bash
    ros2 topic pub --rate 10 /cmd_vel geometry_msgs/msg/Twist \
    "{linear: {x: 0.05, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.03}}"
    ```

    반대 방향:
    
    ```bash
    ros2 topic pub --rate 10 /cmd_vel geometry_msgs/msg/Twist \
    "{linear: {x: 0.05, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: -0.03}}"
    ```
    
    초기에는 `wz = ±0.03`, 최대 `±0.05` 정도만 쓰는 게 좋습니다.

    ## 6. 지면 테스트 순서
    
    공중 테스트가 괜찮으면 지면에서는 이 순서로 합니다.
    
    ```
    1. STAND 30초 유지
    2. RL 모드 + cmd_vel 0.0
    3. vx=0.03, 2~3초
    4. 정지
    5. vx=0.05, 3~5초
    6. 정지
    7. vx=0.08
    8. vx=0.10
    9. yaw는 마지막
    ```
    
    명령 예시:
    
    ```bash
    # RL 전환
    ros2 topic pub --once /control/behavior/mode std_msgs/msg/String "{data: 'RL'}"
    
    # 0.03 m/s 전진
    ros2 topic pub --rate 10 /cmd_vel geometry_msgs/msg/Twist \
    "{linear: {x: 0.03, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
    ```
    
    정지:
    
    ```bash
    ros2 topic pub --once /cmd_vel geometry_msgs/msg/Twist \
    "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
    
    ros2 topic pub --once /control/behavior/mode std_msgs/msg/String "{data: 'STAND'}"
    ```

## rl모델 프롬프트용

SpotMicro용 RL 보행 정책을 학습 중이다. 기본 구조는 CLASSIC/IK trot reference
위에 RL action residual을 더하는 방식이다. 즉 RL이 관절 목표 전체를 직접 내는
게 아니라, 공유 IK 기준 보행 target에 action_scale=0.25rad residual을 더한다.
현재 exp067 모델은 기본 저속 trot 보행이 가능하고, roll/pitch 안정성과 약한
recovery 성능이 개선된 상태다. exp068은 exp067에서 이어서 주행 중 IMU roll/
pitch 흔들림이나 push 이후 자세 회복을 더 잘하게 만들기 위해 angular
disturbance와 tilt recovery reward를 추가한 실험이다.

그리고 질문에 답하면, 네, 이 모델만으로도 걸을 수 있습니다.
다만 “완전한 end-to-end RL 보행”이라기보다는 IK 기반 기본 보행 + RL residual
보정 정책입니다.

현재 모델의 의미는 이렇습니다.

- CLASSIC과 같은 IK 기준 궤적을 사용
- RL은 그 위에 작은 관절 보정값을 더함
- 그래서 기본 trot 보행은 가능
- exp067 기준으로는 저속 전진 안정성, roll/pitch 안정성, 토크 사용이 꽤 좋아짐
- exp068은 여기에 “전환 복구용 안정화”를 더 학습하는 중

실기체 설명용으로 짧게 말하면:

지금 RL 모델은 단독으로 저속 trot 보행이 가능한 residual policy다. IK trot
reference를 기본으로 쓰고, RL이 각 관절에 ±0.25rad 범위의 보정 action을 얹어서
균형, 접촉, 자세 안정성을 개선한다. 기본 보행 정책으로도 쓸 수 있고, CLASSIC
주행 중 roll/pitch가 흔들릴 때 recovery mode로 전환하는 용도로 확장 중이다.

주의할 점은 하나 있습니다.
이 모델이 “스스로 발 궤적 전체를 생성하는 정책”은 아니므로, 실기체에서 쓰려면
반드시 같은 IK reference, 같은 observation layout, 같은 joint order, 같은
action scale이 맞아야 합니다. 이게 맞으면 이 모델만으로도 걷는 테스트는 가능합
니다.
