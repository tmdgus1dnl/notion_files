### CA

- 사설 ca 생성
    
    
    1. ca.key생성
        
        `openssl genrsa -out ca.key 2048` 
        
        openssl 라이브러리(툴킷) 사용
        
        RSA(키 생성을 RSA 비대칭 키 알고리즘 사용해서)
        
        key 길이를 2048비트(이후 인증단계도 2048비트)
        
    2. ca.crt생성
        
        `openssl req -x509 -new -nodes -key ca.key -sha256 -days 30 -out ca.crt`
        
        ca.key를 이용해 
        sha-256 해시 암호화를 사용하는
        ca.crt(감별기) 생성
        
        x509는 인증서 종류
        
    
    → ca.key는 이후 과정이 끝나는대로 따로(개인 메모리에) 보관
    

### 서버,rpi용 인증서 생성

- 서버 인증서 생성
    
    
    1. server.key 서버 키 생성
        
        `openssl genrsa -out server.key 2048` 
        
        서버가 가지는 개인키 생성
        
        소켓 연결시 최초 핸드셰이크과정에서 사용
        
    2. server.csr = 서버 인증 요청파일 생성
        
        `openssl req -new -key server.key -out server.csr` 
        
        서버신분 증명 및 server.key 사용을 위해 CA(상위 보안그룹)에게
        제출해야하는 임시 요청파일 생성
        
    3. server.crt =서버 신분증 생성
        
        `openssl x509 -req -in server.csr -CA ca.crt -CAkey ca.key -CAcreateserial -out server.crt -days 30 -sha256`
        
        csr을 토대로 CA가 서버를 감별 할 수 있는
        server.crt생성
        
    
- 라즈베리파이 인증서 생성
    
    
    1. raspi.key raspi키 생성
        
        `openssl genrsa -out raspi.key 2048` 
        
        라즈베리파이가 가지는 개인키 생성
        
        소켓 연결시 최초 핸드셰이크과정에서 사용
        
    2. raspi.csr = 라즈베리파이 인증 요청파일 생성
        
        `openssl req -new -key raspi.key -out raspi.csr` 
        
        라즈베리파이 신분 증명 및 raspi.key 사용을 위해 CA(상위 보안그룹)에게
        제출해야하는 임시 요청파일 생성
        
    3. raspi.crt =라즈베리파이 신분증 생성
        
        `openssl x509 -req -in raspi.csr -CA ca.crt -CAkey ca.key -CAcreateserial -out raspi.crt -days 30 -sha256`
        
        csr을 토대로 CA가 라즈베리파이를 감별 할 수 있는
        raspi.crt생성
        
    

### 소켓연결

1. TCP 핸드셰이크 
2. TLS핸드셰이크
- tls 핸드셰이크 과정
    1. 브로커(ec2)에 라즈베리파이가 연결(Client Hello)
        
        보안프로토콜 ,암호화 방식 전달
        
    2. 브로커 응답 (Server Hello)
        
        서버가 먼저 본인 증명을 위해 server.crt전송
        
    3. 서버신분 확인 (Server Verification)
        
        라즈베리파이에 있는 ca.crt로 server.crt 감별
        
    4. 라즈베리파이 신분 요청(Certificate Request)
        
        rpi → 서버로 raspi.crt 전송
        
    5. 라즈베리파이 신분 확인(Client Verification)
        
        서버에 있는 ca.crt로 raspi.crt감
        
    
    1. 대칭키 생성(Client Key Exchange)
        
        인증이 끝난 두 호스트간 현재세션에서 사용가능한
        대칭 키를 생성하여 이후 통신과정에서는 대칭키를 이용해 6계층에서 암호화 하여 전송
        

1. 소켓유지 (keep alive)
