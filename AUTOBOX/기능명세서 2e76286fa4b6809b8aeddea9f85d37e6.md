# 오토박스 기능 명세서

---

## 1. 시스템 개요

### 1.1 시스템 명칭

오토박스 (Autobox) – 스마트 물류 자동 분류 시스템

---

### 1.2 시스템 목적

AI 기반 OCR 인식과 자율주행 분류 장치를 활용하여 물류창고 내 택배 분류 과정을 자동화하고,

웹 대시보드를 통해 **실시간 물류 흐름·장치 상태·통계 정보**를 통합적으로 관리한다.

---

### 1.3 시스템 구성

| 구성요소 | 역할 |
| --- | --- |
| Jetson Nano | OCR 처리, 카메라 입력 기반 인식 |
| Raspberry Pi | 컨베이어 및 분류 장치 제어, 상태 송신 |
| Backend API (FastAPI) | 물류/상태/알림/통계 처리 |
| Web Dashboard | 실시간 모니터링 및 관리 UI |
| MySQL DB | 물류·이력·상태·통계 데이터 저장 |

## 2.1 운송장 관리 기능

> 운송장 식별자 정책
> 
> - `waybill_id` : 내부 API 식별자
> - `tracking_number` : 실제 운송장 번호 (업무 키)

> 운송장 상태 표준값
> 
> - READY / MOVING / COMPLETED / ERROR

---

### 2.1.1 운송장 등록 (스캔 시작)

| 항목 | 내용 |
| --- | --- |
| 기능 ID | WB-001 |
| 기능명 | 운송장 스캔 시작 |
| 설명 | 컨베이어벨트에 박스가 진입하면 새로운 운송장 레코드를 생성한다 |
| 입력 | camera_id |
| 출력 | waybill_id, tracking_number |
| 관련 API | POST /waybills/scan |
| 관련 테이블 | waybill |
| 상태 변화 | 없음 → READY |
| 비고 | tracking_number는 시스템에서 자동 생성 |

---

### 2.1.2 OCR 인식 결과 저장

| 항목 | 내용 |
| --- | --- |
| 기능 ID | WB-002 |
| 기능명 | OCR 인식 결과 저장 |
| 설명 | OCR 인식 결과(배송지, 신뢰도)를 저장하고 스캔 로그를 기록한다 |
| 입력 | waybill_id, recognized_text, destination, confidence_score, camera_id |
| 출력 | 처리 결과 |
| 관련 API | PUT /waybills/{id}/recognition |
| 관련 테이블 | waybill, scan_log |
| 상태 변화 | READY → READY |
| 비즈니스 규칙 | confidence_score < 60 → ERROR 전환 |

---

### 2.1.3 분류 작업 시작

| 항목 | 내용 |
| --- | --- |
| 기능 ID | WB-003 |
| 기능명 | 분류 작업 시작 |
| 설명 | 분류 장치가 박스를 적재하고 목적지 구역으로 이동을 시작한다 |
| 입력 | waybill_id |
| 출력 | 처리 결과 |
| 관련 API | PUT /waybills/{id}/start-sorting |
| 관련 테이블 | waybill |
| 상태 변화 | READY → MOVING |

---

### 2.1.4 분류 작업 완료

| 항목 | 내용 |
| --- | --- |
| 기능 ID | WB-004 |
| 기능명 | 분류 작업 완료 |
| 설명 | 목적지 도착 후 분류 완료 처리 및 처리 시간을 계산한다 |
| 입력 | waybill_id |
| 출력 | completed_at, process_time_sec |
| 관련 API | PUT /waybills/{id}/complete |
| 관련 테이블 | waybill |
| 상태 변화 | MOVING → COMPLETED |
| 처리 로직 | process_time_sec = completed_at − created_at |

---

### 2.1.5 오류 처리

| 항목 | 내용 |
| --- | --- |
| 기능 ID | WB-005 |
| 기능명 | 운송장 오류 처리 |
| 설명 | OCR 인식 실패 또는 시스템 오류 발생 시 오류 상태로 전환하고 알림을 생성한다 |

| 상태 변화 | READY → ERROR |

## 5. 비기능 요구사항

### 5.1 성능

- 대시보드 응답: 2초 이내
- 상태 갱신 주기: 5초

### 5.2 데이터 보관

- 운송장: 90일
- 스캔 로그: 180일
- 통계 데이터: 1년

## 6. 용어 정의

| 용어 | 정의 |
| --- | --- |
| 운송장 | 택배 박스를 식별하는 배송 단위 |
| OCR | 이미지 내 텍스트 인식 기술 |
| 분류 | 박스를 목적지 구역으로 이동시키는 작업 |
| 신뢰도 | OCR 인식 정확도 수치 (0~100) |
