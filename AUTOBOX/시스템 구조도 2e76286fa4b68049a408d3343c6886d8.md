# 시스템 아키텍처 설명

본 시스템은 **컨베이어 벨트 기반 적재 장치(Raspberry Pi)** 와

**자율주행 물류 차량(Jetson Orin Nano)**,

그리고 이를 통합 관리하는 **웹 기반 백엔드 시스템**으로 구성된

**분산형 AIoT 물류 자동화 아키텍처**이다.

각 구성 요소는 역할에 따라 명확히 분리되어 있으며,

실시간 제어, 상태 수집, 사용자 제어를 각각 최적의 통신 방식으로 연결한다.

---

## 1. Web Server (Frontend / Backend)

### 1-1. Frontend (Dashboard)

물류 관리자는 웹 브라우저를 통해 대시보드에 접속한다.

Frontend는 Vue + TypeScript 기반으로 구현되며, 다음 기능을 수행한다.

- 작업 시작/중지 명령 전송
- 현재 처리 중인 박스 수량 및 작업 진행 상황 시각화
- 자율주행 차량 및 컨베이어 상태 모니터링

Frontend와 Backend 간 통신은 **REST API 및 WebSocket**을 사용하여

명령 전달과 실시간 상태 반영을 동시에 지원한다.

---

### 1-2. Backend

Backend는 FastAPI 기반으로 구현되며 시스템 전체의 **오케스트레이션 역할**을 담당한다.

- 작업(Job) 생성 및 제어
- Raspberry Pi에 명령 전달
- 수집된 상태 및 로그 데이터 저장
- 데이터베이스(MySQL) 연동을 통한 이력 관리

Backend는 **라즈베리파이에게만 직접 명령을 전달**하며,

자율주행 차량(Jetson Orin Nano)은 Backend와 직접 제어 관계를 갖지 않는다.

---

## 2. Raspberry Pi (컨베이어 벨트 제어 및 게이트웨이)

Raspberry Pi는 본 시스템의 **현장 제어 노드이자 데이터 게이트웨이**이다.

### 2-1. 물리 제어 역할

Raspberry Pi는 GPIO/PWM 기반으로 컨베이어 장치를 직접 제어한다.

- 적외선/초음파 센서를 통한 박스 감지 (GPIO 입력)
- 컨베이어 벨트 ON/OFF 제어
- 서보 모터를 이용한 박스 밀기 동작

이 모든 동작은 Raspberry Pi 내부의 제어 로직(FSM)에 의해

네트워크 상태와 무관하게 **실시간·안정적으로 수행**된다.

---

### 2-2. Jetson Orin Nano와의 통신

Raspberry Pi와 Jetson Orin Nano는 **TCP Socket** 를 통해 직접 연결된다.

이 통신은 다음과 같은 **실시간 핸드셰이크 및 상태 교환**을 목적으로 한다.

- 자율주행 차량의 주차 완료 여부
- 박스 하차 완료 여부
- 차량 배터리 상태
- 차량 동작 상태(Idle / Moving / Error)

이를 통해 컨베이어 적재 타이밍과 차량 상태를 정확히 동기화한다.

---

### 2-3. Backend로의 데이터 전달

Raspberry Pi는 Jetson에서 수신한 차량 상태와

자체적으로 발생한 컨베이어 이벤트를 **가공·통합**한 뒤

Backend로 전달한다.

- 통신 방식: **TCP Socket**
- 전달 데이터:
    - 작업 진행률
    - 컨베이어 상태
    - 차량 요약 상태
    - 이벤트 및 로그 정보

이를 통해 Backend는 Raspberry Pi 하나만으로

전체 현장 상태를 파악할 수 있다.

---

## 3. Jetson Orin Nano (자율주행 물류 차량)

Jetson Orin Nano는 **인지·판단·이동·하차**를 담당하는

AI 기반 자율주행 노드이다.

### 3-1. OCR 기반 물류 정보 인식

- 카메라로 박스 영상을 입력받아 OCR 수행
- 박스에 기재된 정보를 인식하여 목적지 판단
- 인식된 물류 정보는 내부 판단 로직에서 사용됨

---

### 3-2. 자율주행 및 목적지 이동

- 판단된 목적지를 기반으로 라인트레이싱 또는 자율주행 수행
- 지정된 물류 구역으로 이동
- 도착 상태를 Raspberry Pi에 전달

---

### 3-3. 하차 전용 서보 제어

Jetson에 연결된 모터/서보는 **주행용이 아닌 박스 하차 전용**이다.

- 목적지 도착 후 PWM 신호를 통해 서보 동작
- 박스를 해당 위치로 밀어내는 역할 수행
- 하차 완료 상태를 Raspberry Pi로 전송

---

## 4. 통신 구조 요약

본 시스템은 통신 목적에 따라 방식을 분리하여 사용한다.

| 구간 | 통신 방식 | 목적 |
| --- | --- | --- |
| Backend → Raspberry Pi | REST API | 명령 전달 / 작업 제어 |
| Raspberry Pi → Backend | TCP Socket | 상태·로그 전송 |
| Raspberry Pi ↔ Jetson | TCP Socket | 주차·하차 상태 동기화 |
| Frontend ↔ Backend | REST / WebSocket | 사용자 인터페이스 |

이 구조를 통해

- **실시간 제어는 로컬에서 처리**
- **서버는 관리·기록·시각화에 집중**
    
    하는 안정적인 시스템을 구성한다.
